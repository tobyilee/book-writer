# 개발자를 위한 제텔카스텐 — 옵시디언으로 코드와 글을 함께 키우는 법

> **Logline:** 코드만 쓰던 개발자가 제텔카스텐으로 생각을 연결하고, 코드 리뷰·ADR·TIL이 자연스럽게 블로그·발표·책으로 흘러나오게 만드는 100일짜리 실전 가이드.

## 이 책은 어떤 책인가

소프트웨어 개발자를 위한 Zettelkasten 입문서이자 실전서다. Niklas Luhmann의 9만 장 카드 시스템에서 출발해, Sönke Ahrens의 4가지 노트 분류와 Andy Matuschak의 Evergreen note 원칙을 거쳐, 옵시디언이라는 한 도구 위에서 개발자가 매일 30분으로 굴릴 수 있는 워크플로우로 착지시킨다.

시중의 PKM 책들이 일반 지식 노동자를 향해 있다면, 이 책은 처음부터 끝까지 개발자를 호명한다. 코드 리뷰에서 발견한 패턴, ADR로 남기는 결정, TIL로 쌓는 일일 학습, 회사 vault의 보안 정책, Dataview·Templater 쿼리, Obsidian Git + GitHub Private 저장소 셋업, Claude·Claude Code·NotebookLM과의 결합 — 개발자의 일상에 직접 닿는 통합을 다룬다.

이 책은 Zettelkasten의 학습 효과를 RCT로 보장하지 않는다. 대신 Testing Effect·Spacing·Writing-to-Learn·Extended Mind라는 4가지 인지과학 증거와 한국 개발자 커뮤니티의 실증 사례를 묶어, "독자가 자기 손으로 100일을 실험해 볼 가치가 있다"는 정도까지 책임지고 안내한다. 단정 대신 권유, 처방 대신 실험을 제안하는 책이다.

## 이 책이 필요한 독자

- 노트는 많이 쌓아두지만 다시 안 보는 개발자 — Slack 저장 메시지, GitHub Star, 카카오톡 "나에게 보내기", 벨로그 임시저장 12개에 갇힌 사람
- TIL이나 블로그를 시작했다가 흐지부지된 사람 — 글감은 분명 머리 속에 있는데 글이 되지 않는 사람
- ChatGPT·Claude를 쓰지만 매번 컨텍스트가 사라져 답답한 사람
- 옵시디언은 들어봤지만 어디서 시작할지 모르겠는 사람, 또는 플러그인 30개 깔다가 지친 사람
- 3년차 이상 개발자로서 코드 리뷰·ADR·회고를 개인 지식 시스템에 연결하고 싶은 사람
- 결국 책 한 권, 컨퍼런스 발표 하나, 사내 표준 문서 하나를 자기 손으로 길어 올리고 싶은 사람

옵시디언을 한 번도 써보지 않은 독자도 따라올 수 있도록 30분 셋업 시나리오를 제공하고, 3년차 이상이라면 5장(업무 통합)·6장(AI)·7장(출력)에서 더 큰 수확을 가져갈 수 있도록 두 층위로 설계했다.

## 이 책이 약속하는 것

- 옵시디언 30분 안에 첫 atomic note까지 — 폴더 3개, 단축키 7개, 노트 ID 결정 가이드, "지금은 깔지 마세요" 플러그인 목록까지 포함
- 매일 30분으로 굴리는 Fleeting → Literature → Permanent 워크플로우
- 코드 리뷰·ADR·TIL·회의록을 vault에 통합하는 구체적 패턴 (회사 보안 정책 공존 가이드 포함)
- Dataview·Templater 실전 쿼리 — 결정의 진화 추적, MOC 자동 생성, 주간 리뷰 대시보드
- Claude·Claude Code·NotebookLM을 vault와 결합하는 5가지 자리 — 컨텍스트 파트너부터 RAG까지
- atomic note에서 블로그 → 발표 → 책 한 챕터로 자연스럽게 흘려보내는 출력 단계
- 임계점을 넘기 위한 100일 KPI 프레임 — 좌절 구간을 어떻게 통과하는가
- iOS Drafts·Shortcuts, Android Termux + Fleeting Notes로 모바일 캡처를 데스크톱과 한 흐름으로 잇는 셋업
- 한국 개발자가 자주 빠지는 함정에 대한 명시적 처방 — 카톡 "나에게 보내기" 묘지, 태그 트리 지옥, 한국어 슬러그 처리

이 책이 약속하지 않는 것도 분명히 한다 — Zettelkasten의 학습 효과를 RCT로 보장하지 않는다. 대신 메커니즘 증거와 100일 실험을 위한 척도를 제공한다.

## 차례

1. **우리는 왜 노트를 다시 안 볼까** — 개발자가 빠지는 4가지 함정과 노트 묘지 6가지의 해부
2. **Zettelkasten 한 페이지 요약** — 4가지 노트와 1가지 연결, 30초 컷으로 핵심만
3. **옵시디언 30분 셋업** — 폴더 3개·단축키 7개·노트 ID 시스템·Obsidian Git까지
4. **매일 30분 워크플로우** — Fleeting부터 Permanent까지, "한 시간 읽었으면 두 시간 가공"
5. **개발자의 일상에 녹이기** — 코드 리뷰·ADR·TIL·회고를 vault로 끌어오는 법
6. **AI와 함께** — Claude·Claude Code·NotebookLM이 vault에서 차지하는 5가지 자리
7. **노트가 글이 되고 글이 책이 되는** — atomic → 블로그 → 발표 → 책의 출력 단계들
8. **임계점을 넘기 위한 100일** — 좌절 구간 통과법과 KPI 프레임
9. **당신의 Zettelkasten은 당신을 닮는다** — 도구 너머의 이야기

## 저자 소개

Toby-AI는 AI Book Writers 하네스 위에서 운영되는 AI 저자 페르소나다. 평어체와 청유형, 수사적 질문, 저자의 실패담을 먼저 꺼내는 회고적 어조로 글을 쓴다. 단정보다 권유, 처방보다 실험을 권하는 톤을 일관되게 유지한다.

이 책은 5년간 Notion DB가 죽어가는 모습을 지켜본 한 개발자의 시선에서 출발해, Luhmann·Ahrens·Matuschak·Bob Doto·Nori Parelius의 학술적·실천적 작업을 한국 개발자의 일상으로 번역한다.
