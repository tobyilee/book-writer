# 토이부터 SaaS까지, Vercel과 함께 자라기

*Next.js 개발자가 한 단계씩 올라설 때 Vercel을 어떻게 쓰고, 언제 의심해야 하는가*

> 토이의 첫 배포부터 SaaS의 분기 결정까지, Vercel과 함께 자라며 잘 쓰는 능력과 떠날 수 있는 능력을 동시에 손에 쥐는 4단계 안내서.

- **저자:** Toby-AI
- **버전:** v1.0.0
- **발행일:** 2026-05-06
- **언어:** 한국어 (ko)
- **분량:** 9개 챕터 + 서문·에필로그·부록 4종 / 본문 약 14만 자

## 이 책은 무엇인가

이 책은 한 명의 개발자가 자기 프로덕트를 키우면서 Vercel과 어떻게 관계 맺는지를 4단계로 따라간다. 토이 → 사이드 → 초기 스타트업 → 본격 SaaS. 각 단계는 던지는 질문이 다르고, 켜야 할 가드레일이 다르고, 무서워해야 할 함정이 다르다. 챕터는 기능별이 아니라 **독자의 위치별**로 나뉜다.

기존의 Vercel 자료는 두 갈래로 갈라져 있다. 한쪽은 공식 문서의 거대한 레퍼런스고, 다른 쪽은 청구서 폭탄·시크릿 노출·preview 사고 같은 단편적 사고담이다. 그 사이에 비어 있는 자리 — "지금 내 단계에서 무엇을 켜고, 무엇을 끄지 말고, 무엇을 아직 손대지 말아야 하는가" — 를 이 책이 채운다. 단계마다 같은 리듬으로 접근한다. (a) 이 단계의 트리거 신호, (b) 플랜과 한도, (c) 켜야 할 가드레일·운영 패턴, (d) 비용·보안은 어떻게 다른가, (e) 다음 단계로 넘어가는 신호.

후반부의 톤은 한 번 바뀐다. 7·8장은 단계 챕터에 흩어둔 비용·보안 메모를 한자리에 모아 결판을 본다. 함수 과금 3축을 손으로 계산해 보고, 2026년 4월 Vercel 보안 사고를 해부하며 시크릿 회전 절차를 끝까지 따라가 본다. 9장은 의심하는 법이다. Cloudflare·Netlify·OpenNext·자가호스팅까지 대안 매트릭스를 펼치고, `output: "standalone"` 비상 탈출 빌드를 평소에 손에 두는 습관을 보여준다. 비판으로만 닫지 않기 위해 "그래도 Vercel이 빛나는 자리" 한 절로 균형을 맞춘다.

마법으로 칭송하지도, 부도덕한 lock-in으로 묘사하지도 않는다. 두 시선 모두 자기 프로젝트의 의사결정에는 별로 도움이 안 되기 때문이다. 이 책이 약속하는 단어는 **거리감**이다. 비판도 거리감 안에서, 옹호도 거리감 안에서.

## 누구를 위한 책인가

JS/TS·React·Next.js의 기본은 알지만 Vercel 운영 깊이는 얕은 개발자를 위한 책이다. 다음 중 하나라도 자기 얘기 같다면, 이 책이 도움이 될 가능성이 높다.

- `vercel deploy` 한 줄로 첫 배포는 해봤지만, 환경 변수·도메인·preview·청구서 앞에서는 어디부터 손대야 할지 모르겠다.
- Hobby와 Pro 사이에서 망설이고 있다. 카드를 등록할지 말지, 광고 한 줄 붙여도 되는지가 헷갈린다.
- 트래픽이 만 단위를 넘기 시작했고 청구서가 흔들린다. 어디부터 줄여야 할지 잡히지 않는다.
- 첫 엔터프라이즈 고객이 SOC 2 보고서를 요구하기 시작했다. Enterprise로 가야 할지, 자체 인프라로 옮겨야 할지가 묵직하게 걸린다.
- AI 기능을 붙이고 싶은데, AI Gateway가 뭐고 Fluid compute가 뭔지부터 모르겠다.
- 다른 플랫폼으로 옮길 수 있는 능력은 평소에 어떻게 길러두는지가 막막하다.

다 읽고 나면 자기 프로젝트가 어느 단계인지 진단할 수 있고, 그 단계의 가드레일을 켜고 있고, 청구서 폭탄·시크릿 노출·preview 사고를 미리 시뮬레이션해 본 경험이 있고, 다음 단계로 넘어가는 신호를 알아볼 수 있고, 비상 탈출 옵션을 손에 쥐고 있는 상태가 된다.

## 무엇을 얻게 되는가

- **자기 좌표를 찍는 8축 진단 체크리스트** — PV·결제 여부·팀원 수·시크릿 민감도·cron 횟수·이미지 처리량·글로벌 사용자 비중·AI 호출 여부 — 5분 안에 자기 단계가 잡힌다.
- **단계별 가드레일 체크리스트** — 토이의 5가지(Sensitive·robots·Deployment Protection·Git 자동 배포·기본 도메인), 사이드의 Spend Management·환경별 시크릿 분리·Edge Middleware, 스타트업의 외부 CDN·Cloudflare 앞단·WAF/BotID, SaaS의 Enterprise 협상 준비물.
- **함수 과금 3축 손계산** — Invocations·CPU-hour·GB-hour를 직접 계산해 보고, Fluid의 I/O 대기 무료가 청구서에 만드는 차이를 AI 에이전트 시나리오로 본다.
- **2026년 4월 Vercel 보안 사고 해부** — Sensitive vs non-sensitive 환경 변수의 구조적 차이, 시크릿 회전 + 재배포 절차, preview 환경 폭발 반경 통제.
- **대안 매트릭스와 비상 탈출 빌드** — Cloudflare·Netlify·AWS Amplify·Render·Fly.io·자가호스팅의 한 줄 매트릭스, Next.js 16.2 stable Adapter API가 바꾼 풍경, OpenNext의 위치와 한계, `output: "standalone"` 100-200MB Docker 이미지를 평소에 손에 두는 습관.
- **AI 워크로드를 Vercel에 붙이는 자리 잡기** — AI SDK·AI Gateway의 다중 모델 단일 API 라우팅, Fluid I/O 무료가 AI 에이전트와 LLM 스트리밍에 주는 비용 곡선 변화, Vercel 마진의 *추정 한계* 비판.

## 차례

**1장. 처음 막힌 자리에서 시작하자** — 첫 배포는 왜 그렇게 쉬웠고, 그 다음 한 시간은 왜 막막한가. 책의 4단계 척추와 8축 진단 체크리스트.

**2장. Vercel을 한 번 더 정의해 보자** — Frontend Cloud라는 말 뒤에 묶인 추상화 지도. 빌드 파이프라인, 함수 런타임 3종 개요, 렌더링 전략 4종, Preview Deployment, 그리고 *어느 도구를 어느 챕터에서 깊게 다루는가*의 도구 분담 표.

**3장. 토이 단계 — 무료로 살아남기** — Hobby 플랜의 한도, "비상업적 용도"의 진짜 경계, 토이에서 반드시 켜야 할 5가지 가드레일, 결제 카드를 일부러 등록하지 않는 전술.

**4장. 사이드 단계 — Pro로 올라가는 순간** — Pro 트리거 4가지, 만들자마자 켜야 할 Spend Management, 환경별 시크릿 분리, Sensitive 플래그, Cron Jobs 다회 활용, OG Image 동적 카드.

**5장. 초기 스타트업 단계 — 한도가 처음 흔들릴 때** — 비용 곡선이 꺾이는 5대 항목, 청구서 폭탄 케이스 스터디 두 가지, 외부 CDN·Cloudflare 앞단·DB 분리의 외부 보강 3종, Vercel AI SDK·AI Gateway로 AI 기능 붙이기.

**6장. SaaS 단계 — Pro 한도를 넘는 순간의 결정들** — Enterprise냐 멀티 호스트냐 셀프호스트냐. 함수 런타임 3종 심층 비교(Node·Edge·Fluid), Fluid가 AI 워크로드에 주는 진짜 이득, Enterprise 컨택 시 준비물 체크리스트.

**7장. 비용 — 청구서가 폭발하기 전에** — 단계 챕터에 흩어둔 비용 메모를 한자리에. 함수 과금 3축 손계산, Fluid 시나리오, Bandwidth·Image·Edge Requests 폭증 메커니즘, Spend Management 임계 알림과 자동 일시정지 설정, Attack Challenge Mode.

**8장. 보안 — 시크릿·Preview·사고에서 살아남기** — 환경 변수의 두 종류와 구조적 차이, 2026년 4월 Vercel 사고 해부, 시크릿 회전 절차, Preview 폭발 반경 통제, WAF Managed Rulesets·BotID 메커니즘, 컴플라이언스가 비기술 의사결정에 들어오는 두 시나리오.

**9장. 함정·대안·비상 탈출 — 의심하는 법** — 런타임 한도·Cold Start·ISR 오해·Edge Node API 제약, Lock-in 논쟁 두 관점, Next.js 16.2 stable Adapter API 이후의 풍경, 대안 한 줄 매트릭스, `output: "standalone"` 비상 탈출 빌드, "그래도 Vercel이 빛나는 자리" 한 절로 닫는 옹호.

**부록** — Vercel CLI 치트시트(A) / 단계별 가드레일 체크리스트(B) / 청구서 폭탄 응급 대응 플로우차트(C) / 참고 문헌·URL 모음(D).

## 저자 소개

Toby-AI는 입문자에게 위협적이지 않으면서도 토이로 끝나는 책이 아닌 글을 쓰기 위해 설계된 AI 저자 페르소나다. 평어체 기반의 따뜻한 동반자 톤, 청유형과 수사적 질문을 적극 활용하는 호흡, 마법과 lock-in 비판 양쪽에 똑같이 거리감을 두는 시선이 토비 문체의 척추다. 이 책에서는 한 명의 개발자가 자기 프로덕트를 키우는 4단계 여정을 함께 걷는 동반자의 자리에 선다.
