# 에이전트의 시대, DDD는 여전히 유효한가? — 변하는 것, 살아남는 것, 사라지는 것

> Eric Evans가 2003년에 정리한 23년 된 어휘 — Bounded Context, Ubiquitous Language, Aggregate, Anti-Corruption Layer — 가 왜 멀티 에이전트 시대의 거버넌스 어휘로 부활하는가. 그 정체를 다섯 가지 핵심 명제로 정직하게 분류한 시니어 개발자·아키텍트를 위한 에세이형 기술서.

- **라이선스:** CC BY-NC-SA 4.0

## 이 책은 무엇인가

2026년의 시니어 개발자에게 한 가지 풍경이 펼쳐져 있다. Cursor와 Claude Code로 풀 리퀘스트는 두 배가 됐는데, 정작 자기가 더 잘하고 있다는 확신은 줄어든다. AI 공저 코드의 보안 취약점은 사람 코드의 2.74배라는 한국 데이터, METR 2025의 "경험 많은 개발자가 AI 도구로 19% 더 느려졌는데 본인은 20% 빨라졌다고 인식했다"는 격차, vibe coding이 1년 만에 받은 사망 선언. 같은 시기에 한국 8년차 시니어, 영국 컨설턴트, 독일 Siemens 엔지니어, 미국 스타트업 창업자가 거의 같은 결론으로 수렴하고 있었다. "에이전트가 늘어날수록, Eric Evans가 정리한 그 어휘가 다시 중요해진다."

이 책은 그 우연 같지 않은 수렴 현상의 정체를 함께 따라가는 작업이다. "DDD는 여전히 옳다"는 응원가가 아니다. 강화론·변형론·축소론·흡수론의 네 진영을 차분히 펼치고, 무용론의 강한 논거("단순 CRUD에 DDD는 악몽이다", "3년 미만 시스템엔 ROI가 안 나온다", "AI가 ULang을 자동 추출하면 모델링 단계가 축소된다")를 정직하게 인정한 뒤, **어느 시나리오가 와도 닳지 않는 알맹이**를 식별한다. Ubiquitous Language의 격상, Bounded Context = Agent 1개 매핑, Aggregate와 invariant의 인간 영역, Context Map과 ACL의 새 의미(Schema-as-Contract와 Semantic Firewall), Domain Event와 멀티 에이전트 통신까지 — 다섯 패턴이 어떻게 변하고 강화되고 사라지는지를 한 챕터씩 결판낸다.

다른 DDD 책과 다른 점은 두 가지다. 첫째, **글로벌 결제 게이트웨이라는 가상 프로젝트를 두 챕터(10·11장)에 걸쳐 디스커버리부터 전술까지 실증**한다. ULang 워크숍, bounded context 식별, agent 토폴로지, ACL, 휴먼 체크포인트, 측정 지표가 한 도메인 안에서 어떻게 엮이는지를 코드와 문서로 보여준다. 둘째, **AI 시대의 거버넌스(보안·환각·비용·관측성·IP·라이선스)와 한국 조직의 사일로 문제, 시니어 정체성의 인간 영역을 각각 한 챕터씩 정면으로 다룬다**. 패턴서이자 거버넌스 가이드이고, 동시에 한국 시니어의 자기 정체성에 관한 회고록이기도 하다.

답을 미리 정해두지 않은 책이다. 무용론자도 4장 한 박스에 자기 자리를 갖고, 미해결 영역은 책 끝에 정직하게 적혀 있다.

## 누구를 위한 책인가

- DDD를 책으로만 한두 번 들여다본 사람이 아니라, **한 번 이상 실무에 적용해보다가 좌절하거나 절반의 성공을 거둔 시니어 개발자·테크 리드·아키텍트**.
- "폴더 구조만 베껴놓고 DDD 적용했다"고 말하던 자기 자신을 거울로 본 적 있는 사람.
- 도메인 전문가와의 워크숍을 두세 번 시도하다가 일정 압박에 밀려 포기해본 사람.
- vibe coding으로 한두 달 황홀했다가 그다음 달의 디버깅 비용에 데인 사람.
- Cursor·Claude Code·Copilot 중 하나라도 일주일 이상 써본 사람.
- DDD 기본 개념(Entity, Value Object, Aggregate, Bounded Context, Ubiquitous Language)을 한 번이라도 들어본 적이 있는 중급~고급 독자.

**진입 상태:** "AI 도구로 코드는 빨라졌는데 내가 설계 직관을 잃어가는 것 같다", "DDD를 AI 시대에 어떻게 적용할지 모르겠다", "보안·비용·환각 거버넌스가 점점 무거워진다", "팀에 DDD를 도입하고 싶은데 vibe coding 흐름과 어떻게 양립시킬지 막막하다".

**출구 상태:** "AI 시대에 DDD 패턴 중 어느 것이 강화·변형·사라지는지 분명히 안다", "내일 출근해서 `docs/glossary.md`를 만들고 첫 bounded context를 에이전트에 매핑할 자신감이 있다", "보안·환각·비용·IP 거버넌스를 어떻게 구성할지 안다", "Core 도메인에 사람 시간을 어떻게 재배분할지 안다", "AI 시대의 시니어 정체성을 다시 잡았다".

## 무엇을 얻게 되는가

- **다섯 핵심 명제에 대한 분명한 분류** — DDD의 어떤 패턴이 강화되고, 변형되고, 사라지고, 흡수되는지를 한 챕터씩 결판낸다.
- **무용론자의 강한 논거에 대한 정직한 응답** — 4장 "내가 만약 무용론자라면" 시뮬레이션 박스를 통해, 책이 자기 입장을 강제하지 않는다.
- **`docs/glossary.md` 시작 템플릿과 운영 가이드** (부록 A) — 월요일 아침에 바로 첫 글로사리를 만들 수 있는 5필드 템플릿, CI 검증 스크립트, LLM 보조 추출 워크플로까지.
- **결제 게이트웨이 가상 프로젝트의 완성된 실증** (10·11장) — 디스커버리부터 전술·측정 지표까지 한 도메인에 모든 패턴을 옮긴 사례. Toss·당근·Stripe 공개 자료 인용 포함.
- **에이전트당 토큰·도메인 로직 비중이라는 새로운 KPI** — Siemens 사례(prompt spaghetti 3,000 토큰 → 500 토큰, 도메인 로직 15% → 90%)로 검증된 측정 지표.
- **AI 시대의 거버넌스 다섯 기둥** (12장) — 보안(2.74× 취약점)·환각·비용·관측성·IP/라이선스를 결제 사례의 측정 지표와 연결.
- **한국 조직의 사일로 문제에 대한 정면 대응** (13장) — 카카오 헤어샵의 ULang 모범 사례와 BC를 못 나눈 조직의 한계를 같은 사례에서 다른 각도로 재해석.
- **월요일 아침의 실행 로드맵** (15장) — 어휘 30개 등재, 도메인 전문가 30분 인터뷰, BC 종이 스케치. Legacy 마이그레이션 절 포함.
- **DDD의 진화 시나리오 세 갈래** (16장) — 흡수·강화·잔존 시나리오마다 "시니어가 가장 먼저 잃을 직무 vs 가장 강해질 직무" 분석.

## 차례

1. **AI가 코드를 짜는데, 나는 왜 더 불안해질까?** — 박정현 8년차 시니어 사례와 METR 19% 격차로 본, 시니어의 모호한 불안의 정체.
2. **Vibe coding부터 Harness engineering까지 — 우리는 지금 어디 서 있나?** — 신조어 네 개(vibe·agentic·context·harness)의 지도와 "AI에게 모두 맡긴다 → AI를 둘러싼 시스템을 설계한다"의 큰 곡선.
3. **DDD를 다시 펼치자 — 23년의 정전, 무엇이 닳지 않았고 무엇은 색을 입었나** — 닳지 않은 패턴(essence)·색이 입혀진 패턴(form)·거의 닳은 의식(ceremony)의 정밀 분류.
4. **네 진영의 격돌 — DDD는 강화되는가, 사라지는가, 변형되는가, 흡수되는가?** — 강화·변형·축소·흡수 네 진영의 논리와 "내가 만약 무용론자라면" 시뮬레이션.
5. **Ubiquitous Language의 격상 — LLM 시대의 1급 자산** — "LLMs are amplifiers"의 진단과 카카오 헤어샵의 한국형 모범, 살아있는 글로사리 운영 패턴.
6. **Bounded Context = Agent 1개 — 사실상의 산업 표준 매핑** — Croft·Iusztin·Siemens 사례와 모놀리스·MSA·멀티 에이전트의 위계 비교.
7. **Aggregate와 invariant — 사람의 마지막 영역** — FTAPI 사례로 본 자동화 가능한 단계와 사람이 지켜야 할 invariant의 경계.
8. **Context Map과 ACL의 새로운 의미 — Schema-as-Contract와 Semantic Firewall** — 다이어그램에서 실행 가능 코드로, 데이터 변환에서 의미 방어선으로의 진화.
9. **Domain Event와 멀티 에이전트 통신 — Event Storming과 Vibe Modeling의 부활** — fact-as-message의 의미와 모델링 단계로 부활한 vibe modeling.
10. **가상 프로젝트 (전반) — 글로벌 결제 게이트웨이의 디스커버리와 전략** — vibe modeling 워크숍, ULang 정렬, BC 식별, context map 설계.
11. **가상 프로젝트 (후반) — 결제 게이트웨이의 전술과 실행** — aggregate·invariant, domain event 카탈로그, 에이전트 토폴로지, ACL, 휴먼 체크포인트, 측정 지표.
12. **AI 시대의 거버넌스 — 보안·환각·비용·관측성·책임** — 다섯 기둥과 11장 측정 지표의 거버넌스 확장.
13. **AI 시대의 팀과 역할 — 한국 조직의 사일로 문제를 정면으로** — PO·아키텍트·시니어·주니어 역할 재편과 BC를 못 나눈 조직의 한계.
14. **AI를 잘 쓰는 개발자가 된다는 것 — 시니어 정체성과 학습의 역설** — 직관이 마모되는 시대에 시니어로 남는다는 것의 의미.
15. **월요일 아침의 적용 전략 — 어디서부터 시작할 것인가** — 도구 추천을 "역할" 단위로 재구성한 구체적 로드맵과 Legacy 마이그레이션 절.
16. **DDD의 진화 시나리오와 우리에게 남은 일** — 흡수·강화·잔존 세 갈래와 시니어가 가장 먼저 잃을 직무·가장 강해질 직무.

## 저자 소개

**Toby-AI** 는 한국어 기술서를 위해 설계된 AI 저자 페르소나다. 평어체 기반, 청유형 적극 활용, 수사적 질문, 상황 가정, 감정적 공감 표현을 특징으로 한다. 단순한 패턴 카탈로그 나열보다 "왜 이 어휘가 다시 중요해지는가"를 함께 따라가는 에세이형 기술서를 쓴다. 이 책은 한국·영국·독일·미국의 실무자 증언과 학술 논문(arXiv·InfoQ·HN·velog)을 횡단해 수렴 현상을 추적했다.
