# DBA처럼 생각하는 스프링 개발자

> JPA 뒤에 숨어 살던 자바 스프링 개발자가 MySQL을 정면으로 다루는 자리에 서기까지 — DBA가 되지 않으면서도 DBA처럼 생각할 수 있게 되는 13개 챕터.

- **저자:** Toby-AI
- **버전:** v1.0.0
- **발행일:** 2026-05-18
- **언어:** 한국어 (ko)
- **분량:** 약 16개 단위 (서장 + 14장 + 종장) / 본문 약 287,000 자

## 이 책은 무엇인가

MySQL 8.0이 나오고도 한참이 지났는데, 스프링 개발자 안에는 여전히 "JPA가 알아서 해주겠지"로 덮어두는 영역이 있다. `EXPLAIN`을 열어도 어디서부터 읽어야 할지 막막하고, `Repeatable Read`와 `Read Committed`의 차이를 물으면 "기본값이요"라고 답한다. Aurora로 옮기자는 회의에 들어가지만, 왜 옮기는지보다 옮긴 뒤에 뭐가 달라지는지가 더 궁금하다.

이 책은 그 풍경에서 출발한다. DBA가 되라는 책이 아니다. DBA라는 직업은 따로 있고 그 책임은 그 사람들에게 맡기는 것이 옳다. 다만 데이터베이스 앞에 섰을 때 무엇을 물어야 하는지, 어떤 그림을 떠올려야 하는지를 아는 스프링 개발자를 목표로 삼는다 — **DBA처럼 생각하는 스프링 개발자**.

InnoDB 내부의 멘탈 모델에서 시작해 트랜잭션과 락의 메커니즘, 인덱스 설계, `EXPLAIN ANALYZE` 분해, 윈도우 함수·CTE·JSON 인덱싱, 도메인 모델링과 스키마, JPA의 일상 함정과 분산락 패턴, RDS와 Aurora의 분기점, 관측·보안·백업·DR, 메이저 업그레이드까지 — 한 권으로 묶었다. 기준 환경은 MySQL 8.0.30 이상(8.4 LTS 변화 포함), Spring Boot 3.x, Hibernate 6.x다.

본문 곳곳에 우아한형제들, 토스, LINE, 카카오 등 한국 엔지니어들의 공개 경험과 Vlad Mihalcea, Skeema, Percona 같은 글로벌 커뮤니티의 글이 녹아 있다. 이 책의 단단함은 어느 한 사람의 직관이 아니라 여러 사람의 경험이 만든 합의다.

## 누구를 위한 책인가

자바·Spring Boot로 CRUD 애플리케이션을 짜본 중·고급 개발자가 주 독자다. JPA 엔티티를 만들어봤지만 N+1·OFFSET 페이지네이션·배치 INSERT의 함정을 한 번씩은 겪었고, 슬로우 쿼리 알람 앞에서 어디서부터 손대야 할지 막막했던 사람이라면 이 책이 정확히 가려는 자리에 있다.

"Aurora로 옮기자"는 회의에서 결정 근거를 자기 손으로 정리하고 싶은 개발자, 백업은 있는데 복구 리허설은 안 해본 팀의 누군가, 8.0에서 8.4로 업그레이드 일정을 받아들고 체크리스트가 필요한 사람도 이 책이 대상이다.

기본 SQL과 JPA 엔티티 생성 경험이 선수 지식이다. `EXPLAIN`을 본 적 없거나 격리수준이라는 단어가 낯선 독자도 1~3장에서 흡수 가능하도록 설계했다. 특히 2·3·4장(InnoDB 구조, 트랜잭션과 락, 인덱스 설계)은 책의 뼈대로, 뒤의 모든 장이 이 세 장의 그림 위에서 이야기된다.

## 무엇을 얻게 되는가

- InnoDB의 버퍼풀·redo 로그·MVCC·B+Tree를 한 장의 그림으로 이해한다
- `EXPLAIN ANALYZE` 트리를 안쪽 노드부터 읽어 슬로우 쿼리를 혼자 분해한다
- 격리수준과 락 메커니즘을 그림으로 이해하고, 데드락 로그를 5분 안에 짚는다
- JPA의 N+1, 깊은 OFFSET, 영속성 컨텍스트 폭발 같은 일상 함정과 출구를 안다
- 배치 INSERT 100배, 청크 분할 UPDATE, named lock 분산락 패턴을 코드로 손에 쥔다
- RDS와 Aurora의 분기점을 우리 워크로드 기준으로 직접 정리할 수 있다
- 분기마다 PITR 리허설을 돌릴 수 있는 자신감을 얻는다
- MySQL 8.0 → 8.4 메이저 업그레이드 체크리스트를 손에 쥐고 일정을 받는다

## 차례

0. **서장.** ORM 뒤에 숨어 살던 우리에게 — JPA 추상화의 이면, 이 책이 가려는 자리
1. **1장.** MySQL 8.x가 바꾼 풍경 — 8.0 혁신 요약, 8.4 LTS 다섯 가지 디폴트 변경, 다운그레이드 불가의 무게
2. **2장.** InnoDB라는 엔진을 그림으로 이해하자 — 버퍼풀·redo 로그·MVCC·B+Tree·AHI·Change Buffer 한 그림
3. **3장.** 트랜잭션, 격리수준, 그리고 락의 메커니즘 — 4가지 격리수준, 갭·넥스트키·레코드 락, 데드락 로그 읽기
4. **4장.** 인덱스 설계는 데이터 구조 설계다 — 클러스터링·세컨더리·복합·커버링·Invisible 인덱스, 쓰기 비용까지
5. **5장.** EXPLAIN ANALYZE를 안에서 바깥으로 읽기 — 트리 탐색법, 인덱스 비사용 원인, filesort·temporary 해석
6. **6장.** SQL의 표현력을 다시 끌어올리자 — 윈도우와 CTE — 집계·순위·비교·재귀 쿼리 실전 패턴
7. **7장.** JSON 데이터 모델링과 인덱싱 — JSON_TABLE, 생성 컬럼 인덱스, 다중값 인덱스
8. **8장.** 도메인 모델링과 스키마 설계의 만남 — 정규화·반정규화 트레이드오프, 소프트 딜리트, 다중 테넌시
9. **9장.** JPA로 MySQL 다루기 — 영속성 기본기와 함정 — N+1, LazyInitializationException, OFFSET 페이지네이션 함정
10. **10장.** JPA를 넘어 — 성능·동시성·락 패턴·배치 — 배치 INSERT·청크 UPDATE·낙관적/비관적락·named lock 분산락
11. **11장.** AWS RDS와 Aurora — 인스턴스 선택·복제·분할의 분기점 — 스토리지 아키텍처 차이, 읽기 복제, 분할 판단 기준
12. **12장.** 운영의 눈 — 관측, 보안, 백업·복구·DR — Performance Schema, 감사 로그, mysqldump·XtraBackup·PITR 리허설
13. **13장.** 통합 사례 — 결제 시스템 한 건을 끝까지 분해해 보자 — 1~12장 도구를 총동원한 실전 분해, 도메인 일반화
14. **14장.** 업그레이드와 그다음 — 메이저 버전을 넘는 법 — 8.0 → 8.4 체크리스트, Blue/Green 배포, Percona pt-upgrade
15. **종장.** 책을 덮으며 — DBA처럼 생각하는 자리에 섰다는 것의 의미

## 저자 소개

Toby-AI는 자바·Spring 생태계와 데이터베이스 엔지니어링이 만나는 자리에서 글을 쓰기 위해 설계된 AI 저자 페르소나다. 한국과 글로벌 엔지니어링 커뮤니티의 공개 경험을 평어체로 묶어 멘토링형 기술서를 짓는다. MySQL과 PostgreSQL을 비롯한 RDBMS, JPA·Hibernate, 분산 시스템, 운영 자동화에 관심이 깊다. 이 책은 Toby-AI 하네스 v1.2.0으로 저술됐다.
