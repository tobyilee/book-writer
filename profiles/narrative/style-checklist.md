# narrative 스타일 체크리스트

style-guardian이 서사 챕터 초안을 검수할 때 쓰는 항목이다. 원문 인용 → 대안 형식으로 구체 제안. 한 챕터당 5~10건, 과교정 금지.

## 서사 항목

| 항목 | 확인할 것 |
|------|----------|
| 보여주기 vs 말하기 | 감정을 "슬펐다"로 명명했는가? → 행동·신체·대사로 환원 제안 |
| 오프닝 | 작가 요약·배경 설명으로 열었는가? → 장면·훅으로 교체 |
| 시점 일관 | 한 장면에 시점 점프(head-hopping)가 있는가? |
| 대사 기능 | 대사가 정보 떠먹이기(As-you-know-Bob)인가? → 갈등·성격 드러내는 대사로 |
| 대사 태그 | 부사 떡칠("화내며 말했다")인가? → 행동 비트 + "말했다" |
| 감각 디테일 | 장면에 시각 외 감각(소리·촉각·냄새)이 있는가? |
| 페이싱 | 긴장 장면인데 문장이 늘어지는가? / 정적 장면인데 너무 급한가? |
| 씬 완결성 | 인물이 장면 시작과 끝에서 달라졌는가? (정체된 씬) |
| 여백 | 모든 걸 설명해버려 독자 추론 여지가 없는가? |
| 연속성 | 인물 이름·외모·관계·타임라인이 앞 챕터와 어긋나는가? |

## 우선순위 라벨

- **Critical**: 시점 붕괴, 연속성 모순, 장면 부재(전부 요약)
- **Should**: 말하기 과다, 설명 대사, 페이싱 어긋남
- **Nice**: 대사 태그·단어 선택 윤문

> 연속성 모순은 항상 Critical로 보고한다. 소설의 가장 흔하고 치명적인 실패 지점이다. style-guardian은 1차로 눈에 띄는 모순만 잡고, 본격 검수는 `continuity-keeper`가 `story_bible.md` 대조로 수행한다.
