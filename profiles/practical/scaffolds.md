# practical 챕터 스캐폴드

실용서 챕터 유형별 구조다. 핵심은 **독자가 따라 할 수 있는 순서**다. 구조화 블록(목록·메타박스·체크리스트)을 적극 쓴다.

## 1. 레시피·실습형 (요리, DIY, 만들기)

```
[도입] 무엇을 만드는가 + 한 줄 매력 ("바삭한 겉, 촉촉한 속")
[메타 박스] 소요 시간 · 난이도 · 분량(인분/개수) · 필요 도구
[재료/준비물] 분량·단위 명기한 목록
[단계] 번호 매긴 순서 — 한 단계 한 동작, 이유 한 줄, 위험은 동작 앞에
[완성·플레이팅] 결과 확인법 ("이렇게 되면 성공")
[흔한 실수 / 복구] 자주 틀리는 지점과 되돌리는 법
[응용·변형] 한두 가지 변주
```

## 2. 여행 코스·장소형

```
[도입] 이 지역/코스의 성격과 누구에게 맞나
[메타 박스] 이동 시간 · 예산 · 추천 시기 · 소요 일수
[동선] 순서대로 — 장소 → 이동수단/시간 → 머무는 시간 → 다음 장소
[장소별] 볼거리 + 실용 정보(운영시간·요금·팁) + 한 줄 감상
[먹을 곳·쉴 곳] 근처 식당·숙소 옵션
[체크리스트] 이 코스 준비물 / 사전 예약 필요 항목 (`- [ ]`)
[주의] 현지 사정·계절·안전 고지
```

## 3. 기법·기술 익히기형 (실용 스킬)

```
[도입] 이 기법이 왜 유용한가 (상황 예시)
[원리 최소] 알아야 할 만큼만 짧게
[기본 동작] 단계별 + 사진/그림(![캡션](images/foo.png))
[연습] 따라 할 수 있는 작은 과제
[흔한 실수] 초보가 자주 막히는 지점
[다음 단계] 숙달 후 시도할 것
```

## 4. 가이드·의사결정형 (고르는 법, 준비하는 법)

```
[도입] 어떤 선택·준비 상황인가
[기준] 무엇을 보고 골라야 하나 (항목별)
[옵션 비교] 표로 장단점·가격·적합 대상
[추천 매트릭스] "~라면 A, ~라면 B"
[체크리스트] 결정 전 점검 항목
```

## 5. 종합·플랜형 (전체 일정·코스 묶음)

```
[도입] 전체 그림 (며칠 / 몇 단계짜리 계획인가)
[전체 일정표] 한눈에 보는 표
[일자별/단계별] 각 항목을 위 스캐폴드로 전개
[예산·준비 총정리] 합산 + 최종 체크리스트
```

## 구조화 블록 markdown 규약 (v1.5.0+)

실용서의 메타·재료·단계·콜아웃은 pandoc **fenced div**로 감싸면 EPUB에서 박스로 렌더된다 (번들 스타일시트 `epub.css`가 자동 적용된다). 클래스명을 정확히 지킨다.

**메타 박스** — 시간·난이도·분량/인원·예산 요약:
```
::: meta
**소요 시간** 40분
**난이도** 중
**분량** 2인분
:::
```

**재료·준비물**:
```
::: ingredients
재료

- 양파 1개
- 올리브유 2큰술
:::
```

**단계** — 내부에 번호 리스트:
```
::: steps
1. 팬을 중불로 달군다.
2. 양파를 넣고 5분간 볶는다.
:::
```

**팁 / 주의(안전)** 콜아웃:
```
::: tip
미리 재료를 계량해두면 한결 수월하다.
:::

::: warning
기름이 튈 수 있으니 물기를 완전히 닦고 넣는다.
:::
```

**여행 일정표** — 내부에 표:
```
::: itinerary
| 시간 | 장소 | 이동 |
|------|------|------|
| 09:00 | 성산일출봉 | 도보 |
:::
```

**체크리스트** — pandoc 태스크 리스트:
```
- [ ] 여권
- [ ] 우산
```

규약 주의: fenced div 여는 줄(`::: tip`)과 닫는 줄(`:::`) 앞뒤로 빈 줄을 둔다. 클래스명은 `meta`/`ingredients`/`steps`/`tip`/`warning`/`itinerary` 외에는 스타일이 없다.

**이미지·다이어그램** — 사진·지도·완성 컷은 실제 이미지 markdown으로 넣는다: `![캡션](images/foo.png)`. 파일은 `{slug}/images/`에 둔다. 구조·동선·흐름처럼 그림으로 보여줄 관계도는 ` ```mermaid ` 블록 + 캡션 줄 `그림 N. {설명}`(N은 항목/장마다 1부터)으로 작성해도 된다 — 빌드 스크립트가 `mmdc`로 미리 렌더하며, 없으면 그대로 두고 넘어간다(graceful degradation).

## 공통 가이드

- 단계는 `::: steps` + 번호 리스트, 준비물은 `::: ingredients` 또는 체크박스, 요약 정보는 `::: meta` 박스, 안전 경고는 `::: warning`
- 한 단계 = 한 동작. 두 동작이 섞이면 나눈다
- 분량·시간·단위는 항상 구체적으로. 추상어는 기준 병기
- 위험·실패 경고는 해당 동작 **앞**에
- 분량 가이드: 항목(레시피/장소/기법)당 1,500~4,000자. 길어지면 단계가 너무 잘게 쪼개졌는지 점검
- 사진·지도·그림은 실제 이미지 markdown `![캡션](images/foo.png)`으로 넣는다 (파일은 `{slug}/images/`). 구조·동선 다이어그램은 ` ```mermaid ` 블록 + `그림 N. {설명}` 캡션으로 대체 가능 (빌드 시 자동 렌더, mmdc 없으면 graceful degradation)
