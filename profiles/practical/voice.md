# practical voice — 실용서 문체

요리·여행·DIY·실용 가이드 프로필이다. 핵심은 **독자가 따라 하면 실제로 된다**. 명료함과 안전이 멋보다 우선한다.

## 1. 문체의 정체성

| 축 | 의미 |
|----|------|
| 명료한 안내자 | 모호함은 실패를 부른다. 한 문장 한 동작, 순서가 분명 |
| 행동 지향 | 배경 설명보다 "무엇을, 어떻게" 먼저. 이론은 필요한 만큼만 |
| 안전·실패 대비 | 다치거나 망칠 수 있는 지점을 미리 경고한다 |
| 친근한 격려 | 초보도 해낼 수 있다는 톤. 겁주지 않되 정직하게 |

## 2. 지시형 문체

실용서는 독자에게 직접 말한다. 평어체 해설과 지시형 단계를 섞는다.

- **단계 지시:** "팬을 중불로 달군다", "왼쪽 골목으로 들어선다" — 동사로 끝나는 명확한 동작
- **이유 한 줄:** 왜 그렇게 하는지 짧게. "센 불이면 겉만 타니 중불로." 이유를 알면 응용·복구가 된다
- **시점·분량 명시:** "10분", "소금 한 자밤", "도보 15분" — 추상어("적당히") 최소화, 쓰면 기준 병기 ("적당히(국물이 자작해질 때까지)")

## 3. 안전·실패 대비

실용서의 신뢰는 여기서 갈린다.

- **위험 경고는 동작 앞에:** "기름이 튀니 물기를 완전히 닦고 넣는다" — 사고 난 뒤가 아니라 전에
- **흔한 실수 박스:** 단계마다 초보가 자주 틀리는 지점을 짚는다
- **복구법:** "너무 짜졌다면 감자를 넣어 끓인다" — 망쳤을 때 돌아오는 길
- **개인차·현지 사정 고지:** "오븐마다 화력이 다르니 색을 보며 조절", "현지 사정으로 운영시간이 바뀔 수 있으니 확인"

## 4. 구조화 블록 (실용서의 뼈대)

산문보다 구조가 정보를 더 잘 나른다. (EPUB 렌더링 심화는 로드맵 P3 — 지금은 markdown 규약으로)

- **재료/준비물 목록:** 분량·단위 명기한 리스트
- **단계 리스트:** 번호 매긴 순서, 한 단계 한 동작
- **메타 박스:** 소요 시간·난이도·인분/인원·예산 등 한눈 요약 (인용블록 `>` 활용)
- **체크리스트:** 여행 준비물·사전 점검은 체크박스 `- [ ]`
- **팁/주의 콜아웃:** `> 💡 팁:` `> ⚠️ 주의:` 형태로 본문과 구분

## 5. 해서는 안 될 것

| 피해야 할 | 대안 |
|----------|------|
| 모호한 분량 ("적당히", "약간") | 기준 병기 ("적당히(반 컵 정도)") |
| 위험 경고를 사고 후에 | 동작 지시 앞에 배치 |
| 검증 안 된 단정 ("무조건 된다") | "대개 ~된다. 안 되면 ~" |
| 배경 설명 장황 | 필요한 만큼만, 단계 먼저 |
| 현지·개인차 무시한 단정 | "환경에 따라 다를 수 있다" 고지 |

## 6. 첨삭 예시

### 원문 (모호·위험 누락)
> 고기를 적당히 구운 뒤 소스를 넣고 좀 더 익히면 완성이다.

### 실용서 스타일로
> 달군 팬에 고기를 올리고 센 불에서 한 면당 2분씩 굽는다. 겉이 노릇해지면 불을 중불로 줄이고(센 불이면 소스가 탄다) 소스를 두 큰술 두른다. 소스가 걸쭉해질 때까지 1~2분 더 익히면 완성이다.
>
> ⚠️ 소스를 넣을 때 기름이 튈 수 있으니 한 발 물러서서 두른다.
