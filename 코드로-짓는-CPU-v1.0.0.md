# 코드로 짓는 CPU

> Kotlin 30줄로 시작해 SAP-1·SAP-2를 거쳐 자기만의 8비트 RISC를 직접 설계하기까지 — 15주 동안 CPU의 안쪽을 손으로 짓는 책.

- **저자:** Toby-AI
- **부제:** Kotlin으로 SAP-1에서 8비트 RISC까지
- **버전:** v1.0.0
- **발행일:** 2026-05-13
- **언어:** 한국어 (ko)
- **분량:** 15개 챕터 / 5부 / 부록 4개 — 본문 약 30만 자
- **라이선스:** CC BY-NC-SA 4.0

## 이 책은 무엇인가

이 책은 "CPU가 어떻게 동작하는가"를 묻는 질문에 코드로 답하는 책이다. 학부 시절 한 번쯤 외운 fetch-decode-execute라는 단어가 자기 코드의 어느 줄에 매핑되는지 손으로 가리킬 수 있게 되는 것이 목표다. Malvino가 *Digital Computer Electronics*에서 제시한 SAP-1 (Simple-As-Possible)에서 출발해, 16바이트 메모리·5개 명령어라는 가장 단순한 골격을 Kotlin으로 모듈 하나씩 조립한다. 그 다음에는 같은 CPU를 SAP-2로 진화시키며 조건 분기·서브루틴·I/O·마이크로코드·디버거가 왜 필요했는지 직접 부딪힌다. 후반부에서는 1980년대 RISC 논쟁을 거쳐 자기 자신의 8비트 RISC ISA를 합성 설계하고, 마지막에는 instruction-accurate 시뮬레이터를 cycle-accurate로 리팩토링하며 "정확도"라는 단어가 한 가지 의미가 아니라는 사실을 손에 익힌다.

nand2tetris가 NAND 게이트에서, Ben Eater가 브레드보드에서 출발했다면, 이 책은 **Kotlin 코드와 테스트**에서 출발한다. 모든 챕터는 GitHub 저장소의 한 커밋으로 끝난다 — 추상적인 도식이 아니라 돌아가는 코드와 그것을 검증하는 테스트가 한 챕터의 보상이다. ADC/SBC overflow 플래그처럼 입문자가 가장 자주 막히는 지점은 정면돌파하면서 그 안에 테스트 주도 개발 전략(kotest, table-driven test, test ROM) 전체를 녹였다.

만들어 봐야 안다는 한 줄을 닻으로 삼는 15주의 산행이다. 매 챕터마다 GitHub 저장소가 한 단계 성장하고, 매 부의 끝에는 정서적 절정 — 첫 미니 CPU, 첫 어셈블러, 자기가 만든 RISC ISA로 짠 Fibonacci — 이 기다린다.

## 누구를 위한 책인가

- **Kotlin/Java 중급 개발자.** 람다·확장 함수·sealed class·when 디스패치 같은 Kotlin 도구를 안다. 다만 *컴퓨터 자체*가 어떻게 동작하는지에 대한 손 감각이 비어 있다고 느끼는 사람.
- **컴퓨터 아키텍처를 학부 1학년 수준에서 다시 짓고 싶은 사람.** "ALU? 마이크로코드? 어셈블러? 다 다른 세계인 줄 알았다"는 인식에서 출발해, 이 모두가 한 권의 책에서 한 줄기로 흐른다는 것을 느끼고 싶은 사람.
- **GitHub의 emulator 코드를 펼쳐도 어디서부터 읽어야 할지 모르던 사람.** 이 책을 다 읽고 나면 ksim65 같은 실제 8비트 시뮬레이터를 펼쳐도 `when` 디스패치와 sealed class 패턴이 무엇을 하는지 즉시 읽힌다.
- **RISC vs CISC가 면접 단어에 불과했던 사람.** 1980년대의 격렬한 논쟁이 왜 격렬했는지, 그 결론이 어떻게 오늘날 Apple Silicon으로 흘러왔는지 따라가 보고 싶은 사람.

읽고 나면 자기 GitHub에 직접 짠 8비트 RISC 시뮬레이터 저장소가 남는다. 그리고 CPU에서 메모리 시스템·I/O·인터럽트·커널·OS로 이어지는 다음 12개월의 청사진이 머릿속에 있다.

## 무엇을 얻게 되는가

- fetch-decode-execute 사이클을 Kotlin 코드의 어느 줄이 무엇을 하는지 단위로 설명할 수 있다.
- SAP-1 (5 명령어·16바이트)을 Kotlin으로 모듈 단위로 조립하고, 자기가 짠 어셈블러로 어셈블한 프로그램을 자기 CPU에서 돌릴 수 있다.
- ADC/SBC의 carry/overflow 플래그가 왜 입문자의 무덤인지, 그리고 어떻게 테이블 기반 테스트로 한 번에 잡는지 손에 익힌다.
- SAP-2 (64KB·확장 ISA)로 진화시키며 조건 분기·CALL/RET·I/O 포트·인터럽트·마이크로코드·디버거를 직접 짠다.
- 8080·6502·Z80·8086의 같은 Fibonacci 코드를 한 페이지에서 비교하며 CISC 계보의 끝을 본다.
- 1980년대 RISC 혁명의 핵심 논점을 추적하고, 자기 자신의 8비트 RISC ISA를 인터럽트 벡터 결정까지 포함해 설계한다.
- instruction-accurate에서 cycle-accurate로 시뮬레이터를 리팩토링하며 "정확도"라는 단어의 여러 층위를 손으로 체험한다.
- 매 챕터의 GitHub 커밋이 학습의 흔적으로 남는다 — 책을 다 덮을 때 자기 저장소에 15개의 단계별 태그가 박혀 있다.

## 차례

### 1부. 출발

1. **컴퓨터는 왜 이런 모습인가** — 1945년 von Neumann 보고서가 정한 골격, Harvard와의 짧은 대비, 왜 8비트로 시작하는가, 그리고 다음 챕터의 30줄 미리 보기.
2. **fetch-decode-execute를 Kotlin 30줄로** — PC→MAR→MDR→CIR로 흐르는 CPU의 가장 작은 심장 박동을 ADD/HLT 두 명령어와 4바이트 메모리로 짠다. 첫 GitHub 커밋.

### 2부. SAP-1 짓기

3. **SAP-1의 청사진과 첫 모듈** — Malvino의 SAP-1을 한눈에 보고, 클록과 프로그램 카운터를 sealed class로 모델링한다.
4. **레지스터와 8비트 산술** — 작은 책장 같은 레지스터부터 시작해 2의 보수·가산기·감산기를 짓고, `Int` + 마스킹 vs `UByte`의 정면 결정을 내린다.
5. **overflow라는 무덤** — ADC/SBC의 carry/overflow 플래그를 Ken Shirriff의 한 줄 공식과 table-driven 테스트로 한 번에 잡는다. 이 챕터에서 테스트 주도 emulator 개발 전략 전체가 도입된다.
6. **메모리·버스·컨트롤러** — 분리된 모듈들이 어떻게 하나의 CPU가 되는가. 컨트롤러가 다른 모듈에 명령을 내리는 첫 순간, SAP-1이 처음 돌아간다.
7. **어셈블러와 디스어셈블러** — `LDA 9 ; ADD A ; OUT ; HLT`가 5바이트의 0과 1이 되는 길, 그리고 그 역방향. 사람의 글이 기계어가 되는 자리.

### 3부. SAP-2로 진화하기

8. **16바이트의 한계** — SAP-2는 왜 "단순 확장"이 아니라 "근본 재설계"인가. 64KB 메모리와 확장 ISA의 의의.
9. **조건 분기·서브루틴·I/O** — `if`와 함수 호출이 기계어 수준에서 어떻게 살아 움직이는가. 인터럽트와의 첫 만남, 메모리 매핑 I/O vs 별도 포트 공간.
10. **마이크로코드와 디버거** — 컨트롤러를 하드와이어드에서 ROM 기반으로 재설계하고, breakpoint·step-over가 있는 디버거를 짠다.

### 4부. 8비트 RISC로 점프

11. **같은 Fibonacci, 네 가지 어셈블리** — 8080·6502·Z80·8086을 같은 계산으로 비교한다. SAP-2가 8080의 친척이었음이 회수되는 자리.
12. **RISC의 약속** — 1980년대의 격렬한 논쟁을 따라가며 "지금까지 짠 SAP-2가 CISC 계보였다"를 인식한다. Apple Silicon으로 흘러온 결말까지.
13. **우리의 8비트 RISC를 설계하다** — 본 책의 합성 8비트 RISC ISA를 인터럽트 벡터 결정까지 직접 한다. SAP-2 버전 Fibonacci보다 짧은 RISC 코드 — 패러다임 전환의 감격.

### 5부. 정확도와 그 너머

14. **instruction-accurate에서 cycle-accurate로** — 지금까지 짠 시뮬레이터를 리팩토링하며 "정확도"의 의미를 손으로 체험한다. 인터럽트 타이밍이 cycle-accurate를 요구하는 진짜 이유.
15. **여기서 어디로** — CPU → 메모리 시스템 → I/O 디바이스 → 인터럽트 → 커널 → OS로 이어지는 다음 12개월의 로드맵. 부동소수점과 문자열의 부재가 무엇을 의미하는가.
