# JVM 출신을 위한 Rust — Spring 다음에 읽는 책

> *Rust는 JVM의 대체가 아니라 무기 추가다. Spring 다음의 시스템을 손에 쥐자.*

## 한 줄 소개 (logline)

Java/Kotlin/Spring으로 다년간 시스템을 만들어온 백엔드 개발자가, Rust로 실무 시스템을 만들 수 있게 되는 16개 챕터의 입문·활용서.

## 어떤 책인가

이 책은 Rust 문법 레퍼런스가 아니다. *JVM 출신만이 알 수 있는 닻*에 새 개념을 묶어 가르치는 *번역 가이드*다. 매 챕터의 끝에는 *JVM 대응물 매핑* 절이 시그니처로 박혀 있다. 16개 챕터 모두에. 빠지는 챕터가 없다는 약속이 책 전체를 한 줄로 꿰는 신호다.

무게중심은 4부로 나뉜다. *왜 Rust인가, 그리고 첫 만남*(Part 1, 1~3장)에서 의심을 호기심으로 옮기고, *Rust의 마음 — 컴파일러와 친구되기*(Part 2, 4~8장)에서 소유권·빌림·라이프타임·트레잇·unsafe까지 책의 가장 두꺼운 영역을 천천히 통과한다. *실무 시스템을 만든다*(Part 3, 9~12장)에서 동시성·async·axum·sqlx로 *"이제 Spring으로 짜던 그 서비스가 Rust로 보인다"*는 자신감에 도달하고, *출시와 그 다음*(Part 4, 13~16장)에서 8MB 컨테이너·OpenTelemetry·JNI·Project Panama·폴리글랏 아키텍처까지 데려간다.

분량은 본문 약 240,000자, 16개 챕터 + 부록 4편. 출판 페이지로 600~700페이지급의 두툼한 활용서다. 다만 *한 챕터에 두 토픽을 욱여넣지 않는다*는 원칙은 끝까지 지킨다 — 7장과 14장이 그 원칙을 본문에 박은 자리다.

이 책은 *마법으로 학습 곡선을 줄여주지 않는다*. corrode의 권고대로 *"4~6개월의 적응 기간"*을 정직하게 가정한다. 이 책의 역할은 그 곡선을 *없애는* 것이 아니라 *함께 걷는* 것이다.

## 누구를 위한 책인가

- ✓ Java/Kotlin/Spring으로 다년간 백엔드를 만들어온 시니어/미들 개발자
- ✓ JVM 생태계는 익숙하지만 Rust는 처음이거나 입문 단계인 분
- ✓ "JVM에서는 X였는데 Rust에서는 Y구나"라는 비교 학습이 가장 잘 와닿는 분
- ✓ 사이드카·hot path·CLI·관측 도구처럼 JVM 옆에 Rust를 얹고 싶은 백엔드 폴리글랏 지망자
- ✗ Rust 문법 레퍼런스만 빠르게 훑고 싶은 분 (이 책은 두툼한 입문·활용서다)
- ✗ JVM을 떠날 결심을 굳힌 분 (이 책은 폴리글랏을 권한다)
- ✗ Rust로 OS 커널·임베디드 펌웨어를 짜고 싶은 분 (이 책의 백엔드 스코프 밖이다)

진입 상태 — JVM 베테랑·Rust 초심자. 출구 상태 — JVM 시스템 옆에 Rust 사이드카·hot path를 직접 만들어 출시할 수 있는 폴리글랏 백엔드 개발자.

## 핵심 약속

1. 4~6개월 적응 기간을 건너 *"Spring 다음의 시스템을 빌드"*하는 데까지 데려간다.
2. 매 챕터마다 JVM 대응물을 명시 — 이미 알고 있는 것에 닻을 내려 새 개념을 흡수한다.
3. borrow checker·라이프타임·async·트레잇처럼 막막한 영역을 *동반자적 톤*으로 통과한다.
4. 부록 4편으로 매핑 표·치트시트·crate 카탈로그·24주 학습 가이드를 손에 쥐어준다.

## 차례

### Part 1. 왜 Rust인가, 그리고 첫 만남

1장. 왜 지금 Rust인가 — JVM 베테랑에게 보내는 초대장
2장. 첫 만남 — cargo로 5분 만에 빌드하기
3장. 변수·타입·함수·모듈 — Java가 알던 모양과 다른 부분

### Part 2. Rust의 마음 — 컴파일러와 친구되기

4장. 소유권 — "한 명만 가진다"는 단순한 규칙
5장. 빌림 — `&T`와 `&mut T`, 그리고 데이터 레이스가 사라지는 이유
6장. 라이프타임 — `'a`라는 메타데이터에 익숙해지자
7장. 트레잇·제네릭·패턴 매칭·에러 처리 — 표현력 도구상자
8장. 스마트 포인터·매크로·unsafe 진입 — 메모리 도구와 안전 경계

### Part 3. 실무 시스템을 만든다

9장. 동시성 기초 — 스레드, channel, Mutex, 그리고 `Send`/`Sync`
10장. async와 tokio — Spring WebFlux/Kotlin Coroutine 다음의 모델
11장. axum으로 첫 HTTP 서비스 — Spring Controller가 Rust로 보이는 순간
12장. 데이터베이스 — sqlx로 컴파일 타임 검증된 SQL을, sea-orm으로 친숙한 ORM을

### Part 4. 출시와 그 다음

13장. 테스트·품질·도구 인프라·CLI — cargo가 IDE·Sonar·JMH·picocli·OWASP를 모두 안고 있다
14장. 출시 — 8MB 컨테이너와 OpenTelemetry 관측
15장. JVM과 함께 — FFI·JNI·Project Panama·폴리글랏 아키텍처
16장. Rust로 가는 길 — 사람·조직·커리어, 그리고 매듭
