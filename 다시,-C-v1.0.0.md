# 다시, C — Apple Silicon에서 ARM 커널까지

> K&R 시절 C를 한 번 떠났다가 돌아온 손에게, 2026년의 C와 Apple Silicon Mac, 그리고 Raspberry Pi 4 위에서 깨어나는 자기 커널까지 한 권으로 잇는다.

## 이 책은 무엇인가

K&R `The C Programming Language`로 C를 배웠다가 Java, JavaScript, Python으로 옮겨가 오래 머문 개발자가, 지금 다시 C로 돌아오려 할 때 마주치는 거리감을 정직하게 좁히는 책이다. 일반적인 C 입문서가 변수와 반복문에서 시작한다면, 이 책은 그 자리를 건너뛰고 "당신이 K&R에서 알던 것 중 무엇이 여전히 유효하고 무엇이 깨졌는가"에서 출발한다. C89·C99·C11·C17·C23이라는 여덟 표준의 지도를 한 장으로 정리하고, Apple Silicon Mac의 모던 개발 환경(Xcode CLT, Homebrew, LLVM, clangd, lldb, QEMU, 베어메탈 크로스 컴파일러)을 한 시간 안에 손에 쥐게 한다.

본문은 포인터·메모리·정의되지 않은 동작(UB)·헤더와 링커·ABI 같은 C의 본질을 다른 언어 출신의 멘탈 모델 위에 다시 얹는다. GC가 없는 세계의 누수·이중 해제·use-after-free를 어떻게 약속과 도구로 다스리는지, 컴파일러가 왜 UB 코드를 통째로 지워버리는지, strict aliasing과 엔디언이 어디서 함정이 되는지 — 정직하게 보여준다. 그리고 그 위에 ASan/UBSan을 정점에 둔 빌드 매트릭스라는 안전망을 짓는다.

이 책의 정점은 베어메탈이다. 후반 다섯 장(Ch11~Ch15)에서 freestanding C의 의미를 짚고, QEMU virt 보드 위에서 첫 "hello, aarch64"를 찍는다. 그다음 `boot.S` 한 줄, 링커 스크립트 한 섹션을 모두 해부한 뒤, 마침내 진짜 Raspberry Pi 4 SD카드에서 자기 커널이 OS 없이 깨어나는 두 번째 정점에 도달한다. 인터럽트·MMU·태스크 스위치라는 다음 걸음으로 가는 짧은 작별 챕터로 마무리한다.

C 입문서도, OS 교과서도, ARM 매뉴얼도 따로따로는 많지만, "옛 C 손이 모던 도구로 다시 C를 손에 익히고, 그 손으로 진짜 ARM 보드를 부팅시키기까지"를 한 줄로 잇는 책은 흔치 않다. 이 책은 그 비어 있던 다리를 한 권으로 채운다.

## 누구를 위한 책인가

- **K&R 시절 C를 만났던 옛 C 경험자.** 이후 Java·JavaScript·Python 같은 메모리 안전 언어에서 오래 일했고, OS·임베디드·런타임·게임 코어 같은 영역으로 다시 내려가고 싶은 사람.
- **"포인터·UB·링커가 다시 무섭다"는 솔직한 두려움을 가진 개발자.** 변수와 반복문 설명은 필요 없지만, 모던 C의 함정과 안전망을 체계적으로 다시 익히고 싶은 사람.
- **베어메탈·운영체제·임베디드로 가는 다리를 찾는 개발자.** Raspberry Pi나 QEMU 위에서 자기 커널을 한 번 깨워보고, 그 경험을 발판으로 인터럽트·MMU·태스크 스위치로 넘어가고 싶은 사람.
- **Apple Silicon Mac을 주 개발 머신으로 쓰는 사용자.** ARM Mac에서 AArch64 크로스 컴파일·QEMU 가속이 사실상 네이티브로 도는 이 시점의 이점을 충분히 활용하고 싶은 사람.

진입 상태는 "K&R 시절의 흐릿한 C 기억과 모던 도구·표준에 대한 막막함"이고, 출구 상태는 "2026년 표준 지도와 sanitizer 안전망을 손에 넣고, QEMU와 진짜 라즈베리 파이 위에서 자기 커널이 부팅되는 화면을 본 자리"다.

## 무엇을 얻게 되는가

- **2026년 C 표준 지도.** C89에서 C23까지 여덟 표준의 핵심 변화와, 새 프로젝트·기존 코드베이스·베어메탈 각각에 어떤 `-std=`를 골라야 하는지에 대한 명확한 기준.
- **Apple Silicon Day 0 셋업.** Xcode CLT, Homebrew, 모던 LLVM, GCC, AArch64 베어메탈 크로스 컴파일러, QEMU까지 한 시간 안에 완성하는 체크리스트와, VS Code + clangd·Neovim·CLion·Zed의 솔직한 비교.
- **포인터·메모리·UB의 직관 복구.** 다섯 시나리오로 손에 다시 익히는 포인터, malloc/free·소유권 약속·누수와 이중 해제, UB와 strict aliasing이 코드를 통째로 지우는 이유까지.
- **ASan/UBSan을 중심에 둔 안전망.** 정적 분석·포매터·libFuzzer를 위성으로 둔 빌드 매트릭스. 일상 개발 빌드에 무엇을 켜 둬야 하는지, 어떤 옵션이 어떤 버그를 잡아내는지 손에 익히는 챕터.
- **헤더·링커·ABI의 손에 잡히는 모델.** `.c`가 `.o`가 되고 한 실행파일로 묶이는 사이클, 함수 호출이 실제로 어떻게 일어나는지, AArch64 ABI의 다리를 미리 놓아 베어메탈 챕터의 부담을 줄인다.
- **freestanding C의 의미와 베어메탈 첫걸음.** `-ffreestanding`이 무엇을 잃고 무엇을 남기는지, `printf` 없는 세계에서 직접 부트스트랩을 짜는 의미.
- **QEMU에서 첫 "hello, aarch64".** 네 파일(`boot.S` + `kernel.c` + `linker.ld` + `Makefile`)이 어떻게 협력해 첫 글자에 도달하는지, 한 줄씩 한 섹션씩 해부.
- **진짜 Raspberry Pi 4 부팅.** SD카드 부트 파티션·UART 시리얼 케이블·실제 보드 위에서 자기 커널이 깨어나는 두 번째 정점.
- **다음 걸음의 길라잡이.** 인터럽트·MMU·태스크 스위치·시스템 콜이라는 OS 커널의 네 단계로 가는 짧은 지도와, 어떤 책·튜토리얼·1차 문서를 다음에 펼쳐야 할지.

## 차례

1. **다시 C를 만나기 전에 — 추억과 현재 사이** — 왜 지금 다시 C인가, K&R 시절과 지금의 C는 무엇이 같고 무엇이 변했는가.
2. **표준의 미로 — C89에서 C23까지** — 여덟 표준의 지도, `-std=c17`과 `-std=c23` 중 무엇을 고를지에 대한 실용적 답.
3. **Mac에서 C 개발하기 — Apple Silicon Day 0** — Xcode CLT·Homebrew·LLVM·크로스 컴파일러·QEMU를 한 시간 안에 깔고 PATH에 올린다.
4. **vi를 떠나며 — 모던 에디터·IDE 고르기** — VS Code + clangd를 기본값으로, Neovim·CLion·Zed를 누구에게 어울리는지 솔직하게.
5. **포인터를 다시 마주하다 — 메모리의 별명** — Java 객체 참조와 C 포인터의 거리, 흐릿한 기억을 다섯 시나리오로 복구.
6. **메모리 관리 — 직접 사 와서 직접 돌려준다** — GC 없는 세계의 누수·이중 해제·use-after-free, 누가 free 할 책임을 어떻게 약속하는가.
7. **정의되지 않은 동작 — C가 너를 어떻게 배신하는가** — 컴파일러가 UB 코드를 지워버리는 이유, strict aliasing·비트 폭·엔디언의 함정.
8. **헤더와 링커 — 컴파일 사이클을 다시 이해하기** — `.c`에서 실행파일까지, 함수 호출이 실제로 어떻게 일어나는가. AArch64 ABI 다리를 미리.
9. **안전망 — Sanitizer를 중심에 둔 빌드 매트릭스** — ASan/UBSan을 정점에 두고 정적 분석·포매터·libFuzzer를 위성으로.
10. **표준 라이브러리의 빈자리 — string·io·container 한계와 대안** — List·Map·정규식·네트워킹이 왜 없는지, 빈 자리를 무엇으로 메우는지.
11. **베어메탈로 가는 다리 — Freestanding C와 호스트 없는 세계** — 호스트 OS가 없을 때 C는 무엇을 잃고 무엇이 남는가, `-ffreestanding`의 의미.
12. **첫 AArch64 커널 — QEMU에서 "Hello, ARM!"** — 네 파일이 협력해 첫 글자에 도달하는 큰 그림.
13. **부트 흐름 해부 — `boot.S` 한 줄, 링커 스크립트 한 섹션** — `.bss` 클리어·스택 포인터·primary core 선택·AArch64 ABI까지 한 줄씩.
14. **진짜 ARM — Raspberry Pi 4에서 부팅하기** — SD카드 부트 파티션·UART 케이블·실제 보드 위의 시리얼 출력.
15. **다음 걸음 — 인터럽트·MMU·태스크 스위치로 가는 길** — Hello를 찍은 자리에서 진짜 OS로 가려면 다음에 무엇을 배워야 하는가.

부록 A에는 책 전반에서 쓰인 `example/` 디렉토리의 빌드 정책과 사용법이, 부록 B에는 C 자체·UB·ARM 베어메탈·운영체제·Rust로 다리 건너기·도구와 안전망 갈래의 다음 읽을거리가 정리돼 있다.
