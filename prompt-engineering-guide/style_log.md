# 스타일 로그 — 프롬프트 엔지니어링 가이드 (tech-book / Toby 문체)

활성 프로필: `profiles/tech-book/` (voice.md + style-checklist.md)
검수 대상: `chapters/01_final.md` ~ `13_final.md` (13장 전체)
저술 분담: writer A(1~4) / writer B(5~9) / writer C(10~13)
검수자: style-guardian
검수일: 2026-06-01

전체 인상: 톤 정체성(동반자적 어조·사고 자극·감정적 공감)은 13장 내내 안정적으로 유지된다. 청유형·수사적 질문·감정 단어 모두 band 안에서 움직이고, tic 과포화 없음. 가장 큰 통권 결함은 (1) bold 강조 밀도가 writer A(장당 3개)와 writer B/C(장당 15~25개) 사이에서 시각적 리듬이 다르게 읽히는 것, (2) `### 이 장의 핵심` 선택 규약의 헤딩 레벨 흔들림(수정함). 사실/신선도 항목에서 4장에 미해소 `(사실 확인 필요)` 주석 2건 — fact-checker 라우팅 필요(Critical).

---

## 01장
- 위반 없음. 오프닝(어제 멋진 답→오늘 엉뚱, 실패/경험 장면)·정의("우연을 설계로")·로드맵 클로징 모두 토비 톤에 정확히 맞음. bold 3개(`설계`·`더 나은 프롬프트를 측정으로`)로 절제됨 — 모범적.
- 총평: 책의 문을 여는 장으로 톤·약속·여정 예고가 균형 잡힘. 손댈 곳 없음.

## 02장
- 위반 없음. 정의 뒤집기 오프닝("좋은 프롬프트=긴 프롬프트라 생각한다 / 사실은 정반대")이 1장과 다른 기법으로 잘 변주됨. 일곱 부품 나열을 리스트로 던지지 않고 산문으로 풀어낸 점(23~35행) 토비 문체에 부합.
- 총평: 골격 장인데도 나열식으로 흐르지 않았다. bold 3개로 절제. 깔끔함.

## 03장
- 위반 없음. 상황 가정 오프닝("Claude에 그대로 던졌다고 해보자")·"글자 그대로 읽는다" 비유·prefill 폐기 안내 모두 톤 일관. 휘발성 수치를 부록 A로 빼고 본문엔 원칙만 둔 처리(3행 블록쿼트)가 신선도 규율에 정확히 부합.
- 총평: 모델별 방언 장의 기준점. 절제된 bold(3개)와 친절한 평어체 균형 좋음.

## 04장
### Critical
- [원문, 71행] "…다시 일러주는 편이 안전하다 **(사실 확인 필요 — GPT-5 가이드의 "3~5 메시지마다 재명시" 권고를 GPT-5.5에 외삽한 것)**."
  → **미해소 `(사실 확인 필요)` 주석.** style-checklist 사실·신선도 항목상 leftover 마커는 Critical. 주석 자체는 외삽 사실을 투명하게 밝혀 모범적이나, 최종본에 마커가 남으면 안 됨. **fact-checker가 GPT-5.5 cookbook 대조로 판정·해소하거나, 해소 불가 시 "GPT-5 계열 공통 가이드 기준"으로 마커만 제거하고 외삽임을 본문에 녹여야 함.** style-guardian 재량으로 덮지 않음.
- [원문, 73행] "…이어받게 할 수 있다 **(사실 확인 필요 — GPT-5 계열 API 기능을 GPT-5.5에 적용한 것으로, 5.5에서의 지원 여부·동작 차이 확인 요)**."
  → 위와 동일. `previous_response_id`의 GPT-5.5 지원 여부 확인 후 마커 해소 필요. fact-checker 라우팅.
### Nice
- 오프닝(모순된 두 지시 실패 장면)이 4장에서 매우 효과적. 변경 불필요.
- 총평: 본문 톤은 writer A 4장 중 가장 드라마틱하고 좋음. 단 미해소 사실 마커 2건이 최종본 게이트를 막는다 — 이 둘만 해소하면 통과.

## 05장 (writer B 시작 — 4↔5 경계 점검)
### Should
- [관찰] bold 강조가 15개로 급증(writer A 평균 3개 → 5장 15개). 4→5 경계에서 시각적 리듬이 눈에 띄게 바뀐다. 5장 단독으로는 80단어당 1개 상한 이내(한국어 실단어 기준)라 과밀은 아니나, **통권 일관성 관점에서 writer A 구간과 결이 다르게 읽힘**. 핵심 명제("temperature를 1.0에서 손 떼라" 류) 위주로만 남기고 부수 강조는 평어체 환원 권장. (통권 변주 점검 참조 — 일괄 조정 후보)
### Nice
- [원문, 5행/13행 등] `[웹: Gemini prompting-strategies]` 인라인 출처 표기 — 신선도 규율엔 부합하나 writer A 구간(부록 A 위임)과 표기 스타일이 다름. editor 통합 시 출처 표기 컨벤션 통일 검토.
- 총평: "온도를 건드리지 마라" 단일 메시지를 실패 장면→원리→대안으로 끌고 가는 구성 훌륭. `### 이 장의 핵심` 요약 규약 도입(3~5불릿·결론 압축·새 정보 아님) — 규약 자체는 정확히 지킴.

## 06장
### Should
- [원문, 23·25·27행] 세 연속 문단을 각각 **`추론을 켜는 손잡이`** / **`구조를 나누는 방식`** / **`길이`** bold로 시작 → **기계적 문단-오프너 bold 패턴.** 병렬 구조가 이미 또렷해 bold가 중복. → **수정함**: 세 lead bold 모두 평어체 환원.
- [원문, 88행] 한 문장에 "**비용이 오르고**", "**응답까지 걸리는 시간(지연)도 늘어난다**" 이중 bold → 한 문장 내 과밀. → **수정함**: 둘 다 해제(직후 "올릴수록 똑똑해지지만 느리고 비싸진다"가 이미 핵심을 강조).
- 결과: 25 → 20개. 비교표 셀·"최적 지점" 명제·멀티모델 함정 등 진짜 강조는 보존.
### Nice
- 비교표(11~19행)는 6장의 뼈대로 적절. 표 셀 bold는 정당.
- 총평: 세 모델 종합 장으로 구성·콜백(10~12장 예고) 탄탄. bold 일괄 조정으로 writer A 구간과의 격차를 일부 좁힘.

## 07장
- 위반 없음. 인용 오프닝("AI한테 이메일 써달라고 하면 번역기 돌린 티가 나요") — 13장 중 유일한 동료 인용 오프닝으로 변주 기여. "본보기 하나가 형용사 열 개를 이긴다" 같은 토비식 비유 좋음. bold 18개지만 일상 영역 핵심 주문(역할·맥락·형식) 강조라 정당 범위.
- 총평: 코드 없는 일상 장의 톤 모범. 모델별 보정(96~99행)을 리스트로 짧게 처리한 것도 적절.

## 08장
- 위반 없음. 상황 가정 오프닝("배우는 중이라고 해보자")·"desirable difficulty" 교육학 비유·소크라테스식 구성 모두 톤 일관. bold 11개로 writer B 구간 중 가장 절제됨. 감정 단어 절제(공감 표현 없이도 자연스러움) — 매 절 반복하지 않은 점 오히려 좋음.
- 총평: "정답 자판기 vs 튜터" 대비가 장 전체를 끌고 가는 축으로 명료. 손댈 곳 없음.

## 09장 (writer B 끝 — 9↔10 경계 점검)
- 위반 없음. 정의 뒤집기 오프닝("코드를 더 잘 받는 법이 아니라 결과를 통째로 받는 법") — 2장과 같은 기법이나 거리가 멀어(2↔9) 중복 부담 없음. 9장 가드 블록의 "하지 마라" 사용을 2장 "하라" 원칙과 충돌처럼 보이게 두지 않고 73행에서 직접 해명한 처리 — 토비식 사고 자극의 모범 사례.
- 총평: ReAct 학술 뿌리(90행)로 깊이를 더한 마무리 좋음. `### 이 장의 핵심` 규약 유지. bold 20개지만 가드 블록·핵심 동사 강조라 정당.

## 10장 (writer C 시작 — 9↔10 경계 점검)
- 위반 없음. 짧은 인용 오프닝("이게 더 나은 것 같아.") — 강렬하고 변주 기여. 9→10 경계에서 톤 단절 없음. position bias·rubric·체크리스트 개념을 산문으로 풀고 표 1개로 정리 — 나열식 회피 양호. bold 22개는 평가 개념 키워드(rubric·position bias·성공 기준 등) 강조라 정당 범위지만 writer A 구간 대비 밀도 높음(통권 관찰 참조).
- 총평: 책 심장부의 첫 장. "느낌→점수" 전환을 차분히 끌고 감. 콜백(1장 사이클) 정확.

## 11장
- 위반 없음. 충격 수치 오프닝("사람 평가와 80% 넘게 일치 [Zheng 2023]") — 13장 중 유일한 수치 오프닝으로 변주 기여, 출처 명기로 신선도 규율 부합. 개념/코드 2트랙 분리(〔개발자 트랙〕 표식)는 영리한 구조. mermaid 그림 1개 적절.
### Nice
- [원문, 47행] "끔찍한 일이다" — 감정 단어 적절(11장 유일, 과포화 아님).
- 총평: 책에서 코드가 가장 빽빽한 장인데도 "프롬프트도 코드처럼" 메시지를 잃지 않음. bold 21개는 편향 3종·결정론/judge 등 핵심어 강조로 정당.

## 12장
### Critical (수정함)
- [원문, 101행] 장 끝 요약 헤딩이 `## 이 장의 핵심` (레벨 2) → 5·6·9장은 모두 `### 이 장의 핵심` (레벨 3). **선택 규약의 헤딩 표기 흔들림 = checklist상 흔들림 금지 항목.** → **수정함**: `### 이 장의 핵심`으로 통일.
### Minor (수정함)
- [원문, 48행] 본문에 "여기서 **이 장의 핵심**이 나온다" — 요약 헤딩명과 동일 문구가 인라인 산문에 등장해 충돌. → **수정함**: "여기서 이 장의 승부처가 나온다"로 교체.
### Nice
- 상황 가정 오프닝("두 개를 손에 들고 있다고 해보자") — 3·6·8장과 같은 기법이나 분산돼 부담 없음(통권 점검 참조).
- 총평: 클라이맥스 장으로 작성→평가→비교→선택 루프를 표·mermaid로 완주. 헤딩/인라인 충돌 2건만 정리하면 깔끔.

## 13장 (writer C 끝 — 종합 장)
- 위반 없음. 명제 오프닝("프롬프트는 코드처럼 다뤄라")으로 책 전체를 한 줄에 압축 — 종합 장에 적합. 클로징(여정 회고 + "오늘 프롬프트 하나 골라 다섯 줄 적기" 행동 권유)은 "다음 장에서는~" 공식이 아니라 닫는 매듭 — 마지막 장 처리로 정확. bold 11개로 다시 절제(writer A 수준에 근접).
- 총평: 안티패턴 5종·프롬프트 인젝션 입문·신선도 규율로 책을 안전하게 닫음. 전체 회고가 1장 질문("왜 어떤 날은 엉뚱한 답?")에 답하며 수미상관 — 모범적 마무리.

### 2. 중복·편중 표시

- **오프닝 — 양호.** 상황 가정 = 03·06·08·12 = 4/13 ≈ 31%, 1/3 상한(≈4.3장) **턱걸이 통과**. 단 12장이 4번째라 한 장만 더 늘면 초과되니 추가 장은 다른 기법 권장. 실패 장면(01·04·05)·정의 뒤집기(02·09·13)·인용(07·10)·수치(11)·명제(13)로 폭넓게 분산. **인접 장 같은 기법 중복 없음**(03·04는 상황가정/실패장면으로 다름, 05·06은 실패장면/상황가정으로 다름). ✅
- **클로징 — 양호하나 한 패턴 우세.** "다음 장 예고(브릿지)" = 01·02·03·04·07·08·10·11 = 8장. 다만 매번 단순 "다음 장에서는 ~" 공식이 아니라 **다음 모델/주제의 성격을 한 문장 미리 던지는 변주된 브릿지**라 단조롭지 않음. 요약 박스 클로징(05·06·09)이 사이사이 끼어 리듬 환기. 종합 장(13) 닫는 매듭 정확. ✅
- **tic — 과포화 없음.** "해보자"가 최다지만 장당 1~3회로 band 내. "기억해두자" 장당 0~1회. 감정 단어(난감·찜찜·끔찍·아찔)가 한 장에 몰리지 않고 흩어짐. ✅
- **bold 밀도 — 통권 불균형(유일한 구조적 결함).** writer A(01~04) 장당 3개 ↔ writer B/C(05~13) 장당 11~25개. 단장 기준으론 모두 상한 이내지만, **연속해 읽으면 1~4장은 강조가 거의 없고 5장부터 갑자기 굵은 글씨가 늘어 "다른 사람이 쓴 듯한" 시각적 단차**가 생긴다. 이게 통권에서 가장 또렷한 writer-교체 흔적.

### 3. 장별 재작성/조정 타깃

- **bold 단차 해소 (Should, editor 통합 시 일괄):** 06장은 이번에 25→20으로 선제 조정함. **남은 권장**: ① writer A 구간(01~04)에 핵심 명제 1~2개씩 bold를 *더해* 5장 이후와의 격차를 좁히거나(상향 평준화), ② writer B/C 구간(특히 10·11·12, 21~22개)에서 키워드 나열성 bold를 장당 3~5개 덜어내(하향 평준화). **하향 평준화 권장** — 토비 문체는 강조를 아낄수록 강조가 산다(checklist). 목표 band: 장당 6~12개. editor에게 통합 패스에서 일괄 조정 요청.
- **오프닝 (조정 불필요, 모니터링만):** 현 분산 양호. 단 향후 장 추가/분할 시 상황 가정 오프닝은 더 늘리지 말 것(이미 4장으로 상한 근접). 추가분은 '실패 장면'·'정의 뒤집기'·'수치'로.
- **출처 표기 컨벤션 (Should, editor):** writer A는 휘발성 수치를 부록 A로 위임하고 본문 인라인 출처 거의 없음 ↔ writer B/C는 `[웹: ...]`·`[Zheng 2023, arXiv:...]` 인라인 표기 사용. 둘 다 신선도 규율엔 부합하나 **표기 형식이 통권에서 갈린다.** editor가 통합 시 인라인 인용 표기 형식을 한 컨벤션으로 통일 검토(각주 vs 대괄호 인라인).
