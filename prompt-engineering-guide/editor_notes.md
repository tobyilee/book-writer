# Editor Notes — better-prompts-eval (Phase 4 통합 → Phase 4.5 인계)

editor가 통합 중 발견했으나 **내용·구조 변경은 하지 않고** 기록만 한 사항. Phase 4.5 `manuscript-reviewer`와 (필요 시) 해당 `chapter-writer`에게 넘긴다.

## 통합에서 실제 반영한 것 (표현만 다듬음, 내용 불변)

1. **인라인 출처 표기 통일 (style-guardian Should 반영).**
   - writer B/C 본문에 남아 있던 research-내부 태그 `[웹: ...]`·`[논문: ...]`·`[커뮤: ...]`·`[github.com/...]`·`[Zheng 2023, arXiv:...]`를 **전부 제거**.
   - 학술 인용은 본문 약식 `(저자 외, 연도)`로 통일(예: `(Brown 외, 2020)`, `(Wei 외, 2022)`, `(Yao 외, 2022)`, `(Zheng 외, 2023)`), arXiv ID·URL은 **책 뒤 참고문헌**으로 일원화.
   - 휘발성 수치 출처는 "부록 A 참조"로 통일(5장 Gemini 릴리스일 인용도 `(2026-05-19 발표 — 부록 A 참조)`로 변환).
   - 결과: 본문에 research-내부 태그·arXiv ID 0건(스캔 확인).

2. **bold 밀도 하향 평준화 (style-guardian Should 반영).**
   - 통합 전 편차: writer A(1~4장) 장당 3개 ↔ writer B/C(5~13장) 11~22개.
   - 키워드 나열성·표 echo·코드블록 직전 라벨·"이 장의 핵심" 재강조 bold를 정리. 10·11·12장을 22/21/22 → 13/12/11로 낮춤. 전 장 3~15 밴드로 수렴(표 셀 bold는 정당 강조로 보존).
   - 1~4장(writer A)은 장당 3개로 낮지만, 토비 문체의 "강조를 아낀다" 정신에 부합해 **인위적 추가는 하지 않음**. 통권에서 강조가 과하지 않은 방향으로 통일됨.

## 구조 변경 제안 (editor가 직접 손대지 않음 — 검수자 판단 위임)

- **없음 (구조는 계획대로 견고).** 13장 아크·러닝 예시 연속성·부록 A 격리·이중 트랙(11장)이 02_plan 사양대로 구현됨. 별도 분할/병합 제안 없음.
- 참고: 1·8장이 코드 제외 본문 기준 중앙값의 84%로 가장 짧으나 70% 플로어 위이고 1장은 훅으로 충분 — `length_report.md` 참조. 확장 요청 보류.

## 검수자(Phase 4.5)가 특히 확인할 지점

1. **금지 마커 0건** — `(사실 확인 필요)`·`[리서치 공백]`·`[미완성]` 통합본 스캔 결과 0건(확인 완료). 재유입 없는지 재확인.
2. **사실 일관성 (tech-book 축)** — 부록 A 수치는 `factcheck_log.md`에서 ✅ 판정된 값만 기재(❌ 0건). 본문은 휘발성 수치를 직접 박지 않고 부록 A로 격리 → 장 간 수치 모순 가능성 구조적으로 차단. GPT-5.5/Gemini 버전 디스앰비규에이션 경고가 부록 A·4장·5장에 일관되게 들어감.
3. **교차 콜백 정합** — 2장→3·4장(골격→방언), 6장 비용·지연 저울→10~12장 eval, 6장(환불 요약 "인사")→10·11·12·13장 러닝 예시 연속성 모두 통합본에서 살아 있음(스캔 확인).
4. **부록 A의 "1차 소스 확인" 빈칸** — max output(Claude/Gemini)·cutoff(Claude/Gemini)·일부 가격은 레퍼런스 §0에 확정값이 없어 "1차 소스 확인"으로 표기. 이는 누락이 아니라 **신선도 규율상 의도된 출판 직전 확인 항목**(표 머리 경고 + 체크리스트로 명시). 검수자는 이를 결함이 아닌 정상으로 판단.
