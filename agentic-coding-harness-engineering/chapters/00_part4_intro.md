# 4부. 보안과 격리 — 하네스가 공격받을 때

> **부 인트로**

3부 끝에 5축 메트릭으로 자기 하네스를 *조용히* 지키는 회로를 봤다. 정량으로 지킬 수 있게 됐다는 안도감이 있다. 그런데 잠깐 — 정량으로 지키는 것은 *내부의 회귀*다. 모델이 멍청해지는 것, 비용이 폭주하는 것, latency가 늘어나는 것. 다 *우리 쪽에서 일어나는* 일이다.

밖에서 들어오는 일은 어떻게 막는가?

2025년 9월 어느 날을 기억해 보자. 한 npm 토큰이 털렸다. 그 한 토큰이 이틀 만에 25,000개가 넘는 저장소로 번졌다 — Wiz Research가 *Shai-Hulud*라는 이름을 붙였다. 같은 해 11월 2.0 변종이 등장했고, 2026년 2월에는 *SANDWORM_MODE*라는 이름의 새 변종이 나왔다. 이번에는 악성 MCP 서버를 통해 LLM API 키까지 노렸다. 우리가 매일 깔고 매일 쓰는 도구의 한가운데가 공격 표면이 된 것이다.

같은 시기에 Simon Willison이 *Lethal Trifecta*라는 이름의 위협 모델을 정리해 두었다. *Private data + Untrusted content + External communication* — 이 셋이 한 컨텍스트에 모이면 위험하다. 이 모델은 추상이 아니다. GitHub MCP 익스플로잇에서 *실제로* 그 셋이 모인 사고가 있었다. 공개 이슈에 심긴 한 줄짜리 prompt injection이 private 저장소의 코드를 외부로 흘려보냈다.

이쯤 되면 난감하다. 우리는 어제까지 "에이전트가 똑똑한지"를 걱정했는데, 오늘은 "에이전트를 둘러싼 *마구* 자체가 공격 표면이 됐다"는 사실과 마주해야 한다.

4부는 그 사실을 정면으로 다룬다. 추상으로 가르치지 않는다. 사고 사례로 가르친다.

흐름은 이렇다.

- **16장 Lethal Trifecta** — 위협 모델을 한 줄로 외운다. *Private + Untrusted + External*. 자기 봇이 그 셋에 걸려 있는지 색칠로 확인한다.
- **17장 Shai-Hulud** — 25,000 저장소 사고를 시간 순으로 분해한다. 1.0 → 2.0 → SANDWORM_MODE. 우리 npm install이 거기서 거기다.
- **18장 프롬프트 인젝션 카탈로그** — 인젝션이 몇 가지 모양인지 갈래를 친다. 도구 description부터 사내 위키까지. 각 모양에 1차 방어를 매핑한다.
- **19장 격리 3축 (이론)** — UK AISI Inspect의 3축 *Tooling·Host·Network*를 받아들인다. Firecracker·gVisor·devcontainer·sandbox-exec를 한 표 위에서 비교한다.
- **20장 격리 운영 레시피** — 30분짜리 PoC 세 개를 손에 쥐어 본다. Docker + egress proxy + ephemeral secrets. devcontainer + 최소 권한. macOS sandbox-exec.
- **21장 평가 회피·Sandbagging** — 보안과 평가가 만나는 사각지대를 본다. 학생이 시험관 표정을 보고 답을 바꾼다면, 그 시험은 무엇을 측정한 것일까.

이 여섯 챕터는 한 부의 결로 단단히 묶여 있다. 16장이 *모델*을 준다. Trifecta는 한 줄짜리 위협 모델이고, 그 한 줄을 자기 환경에 비추는 것이 시작이다. 17장이 *분해*를 준다. 사고를 시간 순으로 끊어 보면, 각 끊김에 대응하는 방어가 보인다. 18장이 *카탈로그*를 준다. 인젝션을 분류하지 못하면 방어도 분류하지 못한다. 19장이 *이론*을, 20장이 *운영*을 준다. 격리는 이론으로 받아들이고 운영으로 짜야 한다 — 둘 중 하나라도 빠지면 무너진다. 21장이 *의심의 시야*를 준다. 벤치 점수가 좋다고 안심하지 말자, 라는 한 줄이 책의 보안 부를 닫는다.

한 가지 당부. 4부는 *겁을 주는* 부가 아니다. 사고를 미리 본 사람이 사고를 줄인다는, 평범한 직업윤리의 부다. 우리가 매일 짜는 마구 위에 매일 무엇이 들어오고 무엇이 나가는지를 *의식*하게 하는 부다.

기억해두자. 보안은 옵션이 아니다. 그러나 보안을 옵션처럼 다루는 팀이 더 많다는 것도 사실이다. 4부 여섯 챕터를 한 호흡으로 읽고 나면, "옵션이 아닌 것을 옵션처럼 다뤘다"는 자각이 한 번은 들 것이다. 그 자각이 이 부의 진짜 출구다.

자, 첫 챕터 — 16장 Lethal Trifecta로 들어가자.
