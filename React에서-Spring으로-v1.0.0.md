# React에서 Spring으로

> `fetch`가 때리던 그 블랙박스 안으로 들어가, AI와 함께 Spring 백엔드를 직접 짓고 검증하는 프런트엔드 개발자의 첫 안내서.

- **저자:** Toby-AI
- **버전:** v1.0.0
- **발행일:** 2026-05-25
- **언어:** 한국어 (ko)
- **분량:** 14개 챕터 + 서문·에필로그·부록 / 본문 약 12만 자 (코드·부록 포함)
- **기준 환경:** Spring Boot 3.x · Spring Security 6.x · Java 17/21 LTS

## 이 책은 무엇인가

React나 Next.js로 화면은 능숙하게 만들지만, 서버는 여전히 `fetch`가 때리는 블랙박스처럼 느껴지는 사람을 위한 책이다. 이 책은 그 블랙박스를 열어, 요청 하나가 서버에 도착해 라우팅·검증·처리·응답으로 흘러가는 과정을 프런트엔드 개발자가 이미 아는 언어로 다시 설명한다. Gradle은 `package.json`에, MockMvc는 `supertest`에, ORM은 이미 써본 그 ORM에 빗대며, 낯선 것을 익숙한 것의 반대편으로 뒤집어 본다.

다루는 범위는 Spring MVC와 JPA를 중심으로 한 백엔드 입문 전 과정이다. 첫 엔드포인트를 띄우는 것에서 시작해 DI/IoC, DTO·검증·전역 예외·CORS, JPA 영속성과 N+1 같은 운영 함정, 인증·인가, JWT와 세션 두 방식의 보안, Thymeleaf 서버 렌더링까지 이어진다. 흩어진 토막 예제가 아니라, **'TaskBoard'라는 하나의 할 일 관리 프로젝트**를 빈 폴더에서 인증·화면을 갖춘 서비스로 14장에 걸쳐 키워나가는 연속 서사로 묶었다.

다른 입문서와 구별되는 두 축이 있다. 하나는 **AI 페어코딩을 동반자로 정면에 놓되, "AI는 만들고 검증은 사람이 한다"는 태도를 책 전체에 녹였다는 것**이다. Cursor나 Claude Code가 척척 만들어주는 Spring 코드의 옳고 그름을 스스로 판별하는 눈을 기른다. 다른 하나는 **버전 신선도 함정**이라는 보조 서사다. AI와 인터넷은 종종 한물간 코드를 자신 있게 내민다 — `javax.*`, `WebSecurityConfigurerAdapter`, `authorizeRequests` 같은 것들. 이 함정을 2장에서 개념으로 심고, 10장 보안에서 정면으로 터뜨리고, 13·14장에서 검증과 마이그레이션으로 회수한다.

빠르게 변하는 기술을 다루는 책인 만큼, 기준 시점(2026년 5월)과 버전을 본문 곳곳에 명시했다. 버전 숫자·API 시그니처는 시점에 따라 달라질 수 있으니, 책은 늘 공식 문서를 1차 근거로 함께 확인하는 습관을 강조한다.

## 누구를 위한 책인가

자바 문법과 SQL 기본은 알지만 백엔드 실무 경험은 아직 없는 프런트엔드(React/Next.js) 개발자를 위한 책이다. 특히 Cursor·Claude Code로 AI 페어코딩은 해봤으나, 생성된 Spring 코드가 맞는지 틀린지를 판단하지 못해 답답했던 입문자에게 맞춰져 있다.

- **진입 상태:** 클라이언트는 능숙하게 만들지만 서버는 블랙박스. 백엔드 실무 경험 0. AI가 준 Spring 코드의 옳고 그름을 가리지 못함.
- **출구 상태:** Spring Boot 3.x로 REST API를 직접 설계·구현하고, JPA로 DB에 영속화하고, JWT·세션 두 방식의 보안을 적용해 트레이드오프를 설명할 수 있다. Thymeleaf SSR을 CSR과 비교해 언제 무엇을 쓸지 판단한다. 무엇보다 AI가 준 코드를 읽고, 버전 신선도 함정을 잡아내고, SQL 로그로 동작을 검증할 수 있는 백엔드 입문자가 된다.

순수 백엔드 첫 입문자(자바·SQL을 전혀 모르는 사람)나 이미 Spring 실무 경험이 있는 개발자에게는 결이 맞지 않을 수 있다. 이 책의 출발점은 어디까지나 "프런트는 알지만 서버는 모르는" 자리다.

## 무엇을 얻게 되는가

- 요청 하나가 서버에서 처리되는 전 과정을, 프런트가 이미 아는 HTTP 모델을 뒤집어 이해한다.
- Spring Boot 3.x로 DTO·검증·전역 예외·CORS를 갖춘 제대로 된 REST API를 직접 설계·구현한다.
- JPA로 데이터를 영속화하고, 영속성 컨텍스트·더티 체킹·지연 로딩의 "마법"과 N+1 같은 운영 함정을 SQL 로그로 가시화한다.
- 인증·인가의 필터체인 모델을 이해하고, JWT(stateless)와 세션(stateful) 두 방식을 같은 프로젝트에 적용해 트레이드오프를 설명한다.
- Thymeleaf 서버 렌더링을 CSR과 비교해, 상황에 맞게 SSR/CSR을 선택하는 판단 기준을 갖춘다.
- AI가 준 Spring 코드에서 버전 신선도 함정(javax→jakarta, 제거된 API 등)을 가려내고, 읽기→버전 식별→동작 확인의 검증 리듬을 손에 익힌다.
- 스펙에서 동작까지, AI와 함께 기능 하나를 처음부터 끝까지 완주하는 워크플로우를 경험한다.

## 차례

1. **fetch 너머에 누가 있나** — 매일 쓰는 `fetch`의 반대편, 서버 입장의 HTTP로 멘탈 모델을 뒤집는다.
2. **AI와 함께 배운다는 것** — AI를 학습 동반자로 쓰되 왜 의심해야 하는지, 신선도 함정의 예고편.
3. **첫 엔드포인트** — 빌드 도구(Gradle ≈ package.json) 오리엔테이션부터 동작하는 서버 띄우기까지.
4. **마법 걷어내기** — 3장에서 쓴 코드의 "마법"을 사후에 걷어내며 DI와 IoC가 실제로 하는 일을 본다.
5. **제대로 된 REST API** — DTO·검증·전역 예외·서버 측 CORS·첫 슬라이스 테스트(MockMvc).
6. **메모리에서 데이터베이스로** — 기존 API에 JPA를 점진적으로 붙여 데이터를 영속화한다.
7. **영속성 컨텍스트라는 출석부** — 더티 체킹·지연 로딩·연관관계 매핑과 `@Transactional` 경계의 마법.
8. **운영에서 터지는 것들** — N+1의 정체와 해법, SQL 로그를 켜는 규율과 `@DataJpaTest`.
9. **보안의 기초** — 인증·인가 개념과 필터체인이 요청을 가로채는 새 멘탈 모델, 최소 설정.
10. **JWT로 인증하기** — stateless 구현과 함께 신선도 함정의 클라이맥스를 before/after로 터뜨린다.
11. **다른 길** — 세션(stateful)을 같은 프로젝트에 적용하고 JWT와의 트레이드오프를 비교한다.
12. **서버가 그리는 화면** — Thymeleaf로 화면을 붙이며 SSR vs CSR을 프런트 관점에서 정리한다.
13. **스펙에서 동작까지** — Claude Code로 기능 하나를 스펙→설계→구현→테스트로 완주한다.
14. **여기서 어디로** — 여정을 종합하고 Spring Boot 3.x에서 4.x로의 마이그레이션 전망으로 닫는다.

부록으로 프런트엔드 ↔ Spring 멘탈 모델 대조표, 신선도 함정 체크리스트, REST의 엄밀한 정의와 HATEOAS를 함께 실었다.

## 저자 소개

Toby-AI는 토비 스타일의 한국어 기술 글쓰기를 위해 설계된 AI 저자 페르소나다. 개념을 독자가 이미 아는 것에 빗대 설명하고, 마법처럼 보이는 동작의 안쪽을 끝까지 걷어내 보여주며, "왜"를 먼저 묻는 평어체 문장으로 입문자의 곁을 지키는 목소리를 지향한다. 이 책에서는 프런트엔드 개발자가 서버라는 낯선 땅에 발을 들이는 길을, 익숙한 풍경에서 출발해 한 걸음씩 안내한다.
