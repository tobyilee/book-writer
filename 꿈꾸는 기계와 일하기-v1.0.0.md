# 꿈꾸는 기계와 일하기

## AI 환각을 다루는 개발자의 사고법

> 환각은 LLM의 버그가 아니라 일반화의 동근원이다. 두려워하지 않고 다룰 줄 알면, 그 위에서 자신의 처리량을 몇 배로 끌어쓸 수 있다.

## 이 책은 무엇인가

2023년 ChatGPT 출시 이후 "AI 환각"이라는 단어는 헤드라인의 단골이 되었다. 가짜 판례를 인용한 변호사가 5,000달러 벌금을 맞고, 보안 연구자가 LLM이 환각한 패키지명을 npm에 미리 등록해 3개월에 30,000번 다운로드되는 슬롭스쿼팅 사건이 USENIX Security 2025에 보고되었다. 동시에 같은 분기에 Dario Amodei는 "AI는 이제 인간보다 덜 환각한다"고 선언했고, Anthropic 자체 연구는 agentic coding을 매일 쓰는 개발자의 디버깅 기술이 6주 만에 47% 떨어졌다고 측정했다. 환각은 줄고 있는가, 늘고 있는가, 아니면 우리가 잘못된 질문을 던지고 있는가.

이 책은 그 잘못된 질문을 바꾸는 것에서 시작한다. 환각을 *없앨* 수 있다는 가정 자체가 틀렸다 — OpenAI 2025 paper(Kalai et al.)가 정보 이론적으로 증명했고, Andrej Karpathy는 한 발 더 나아가 "an LLM is 100% dreaming"이라고 말했다. 환각과 일반화·창의성은 같은 high-dimensional vector blending 메커니즘에서 나온다. 인간 뇌의 좌반구도 매일 똑같은 일을 한다(Gazzaniga의 분리뇌 실험, Memory Reconsolidation, Dennett의 narrative self). 그렇다면 우리에게 필요한 것은 환각률을 0으로 만드는 모델이 아니라, *환각이 일어나도 빠르게 잡히는 환경* — 하네스다.

책은 충격 → 이해 → 화해 → 통합 → 실천 → 종합의 여섯 박자로 흐른다. 슬롭스쿼팅 한 사례로 환각이 코드 바깥까지 사람을 다치게 한다는 충격을 받고, 네 층위(학습·압축·정렬·디코딩)와 평가 인센티브의 벽을 통해 "왜 사라지지 않는가"를 이해한다. 환각이 *기능*이라는 도발을 받아들이고, 인간도 같은 게임을 한다는 거울을 본 뒤, capability overhang이라는 정식 개념으로 "왜 같은 모델·같은 시기에 어떤 팀은 throughput이 3배냐"를 풀어낸다. 그 위에서 정형 검증 · 자기 검증 · 외부 grounding · HITL · 자동 검증 루프 · 하네스 엔지니어링까지, 오늘 저녁 자기 워크플로에 한 줄 깔 수 있는 도구 카탈로그를 손에 쥔다. 마지막 장은 Amodei · OpenAI · Gary Marcus · Karpathy의 네 관점을 병기하고, "agentic coding은 함정이다"라는 비판을 정직하게 수용한 뒤 책의 자세를 다시 한 번 정리한다.

## 누구를 위한 책인가

이 책은 **agentic coding · vibe coding을 쓰거나 도입을 고민하는 개발자**를 위해 쓰였다. Cursor, Claude Code, Codex, Devin 가운데 어느 하나라도 한 번 써본 사람. AI가 그럴듯하게 틀린 코드를 뱉어서 답답해본 사람. 옆 팀의 처리량이 두 배·세 배 빨라진 이유가 궁금한 사람.

- **진입 상태:** "Cursor·Claude Code 써봤는데, 환각 때문에 못 믿겠다. 그런데 동료는 잘 쓴다. 뭘 놓치고 있나?"
- **각성 상태:** "환각은 LLM의 결함이 아니라 일반화와 같은 메커니즘이고, 나도 매일 환각하고 있었다. 두려움이 아니라 다루는 법의 문제구나."
- **출구 상태:** "환각을 자동 검증 루프로 가두고, 그 위에서 capability overhang을 끌어쓰는 자기만의 하네스를 그릴 수 있다. 한 챕터씩 적용 가능한 도구 목록을 손에 쥐었다."

코드 예제는 8a · 8b · 9장에 집중되어 있고, 1~7장은 개념·은유·사례 중심이다. 정보 이론·인지신경과학 개념도 등장하지만 모두 비유로 풀어 옮겼다. 중급 개발자가 부담 없이 따라갈 수 있는 난이도다.

## 무엇을 얻게 되는가

- 환각을 "막연한 두려움"이 아니라 *intrinsic vs extrinsic*, *factuality vs faithfulness*, *confabulation*, *fabrication* 네 축으로 분해해 부르는 어휘
- 환각이 만들어지는 네 층위(학습 데이터 · 다음 토큰 예측 · RLHF · 디코딩)와, 모델이 더 커져도 사라지지 않는 구조적 이유의 이해
- 환각·일반화·창의성이 같은 high-dimensional vector blending에서 나온다는 통합적 시야, 그리고 인간 뇌의 좌반구도 같은 일을 한다는 거울
- *capability overhang* — 같은 모델·같은 시기에 어떤 팀은 throughput이 3배인 이유를 설명하는 정식 개념과, 그것을 끌어쓰는 사람의 자세
- 오늘 자기 환경에 한 줄 깔 수 있는 정형 검증 도구 목록 (타입 시스템 · LSP · 컴파일러 · 단위 테스트 · 린터 · sandbox · dry-run)
- 정형 검증 너머의 환각을 거르는 패턴 — Chain-of-Verification, SelfCheckGPT, Semantic Entropy, verification subagent, Best-of-N, RAG, tool use, FACTS Grounding, HITL
- Anthropic 공식 5가지 권장이 위 카탈로그 안에서 어떻게 자리 잡는지의 매핑
- 자동 검증 루프 · parallel agents · prompt → context → harness의 3단계 진화 · 환각을 자산화하는 brainstorming 워크플로
- 책을 덮자마자 작동시킬 첫 한 줄 — "오늘 저녁 내 워크플로에 한 줄 무엇을 추가할까?"

## 차례

1. **꿈꾸는 기계의 사고 — 환각은 어떻게 우리 앞에 도착했는가** — 슬롭스쿼팅과 가짜 판례 사건으로 환각이 코드 바깥까지 사람을 다치게 한다는 첫 충격을 받고, 책의 입장(환각 = 일반화의 동근원)을 선언한다.
2. **환각의 정체 — 단어 하나에 숨어 있던 네 가지 현상** — *intrinsic vs extrinsic*, *factuality vs faithfulness*, *confabulation*, *fabrication*. 동료와 환각 사례를 토론할 때 어느 축이 깨졌는지 짚을 어휘를 손에 쥔다.
3. **왜 발생하는가 — 학습·압축·정렬·디코딩의 네 층위** — 다음 토큰 예측의 본질, 압축의 필연, RLHF가 calibration을 망치는 보상 구조, 디코딩의 확률성. 환각을 *어디서* 잡을지 결정할 지도.
4. **왜 사라지지 않는가 — 평가의 인센티브와 정보 이론의 벽** — OpenAI 2025 paper의 세 명제, "정답 +1, 오답 0, 기권 0" 채점의 우월 전략, 검증 기법이 0%로 못 가는 이유.
5. **환각은 기능이다 — 일반화·창의성과 같은 근원** — 환각 없는 LLM은 검색엔진이다. High-dimensional vector blending이 일반화·창의성·환각을 같은 메커니즘으로 만든다는 정식 풀이.
6. **인간도 매일 환각한다 — 분리뇌·기억·서사적 자아** — Gazzaniga의 닭발과 삽, Memory Reconsolidation, Dennett의 narrative self. LLM 환각을 *기계의 결함*에서 *지능의 보편 현상*으로 옮기는 거울.
7. **환각 너머의 능력 — Capability Overhang** — Jack Clark의 정의, Karpathy의 imagination bottleneck, Sholto Douglas의 context bottleneck, LessWrong의 Coding Overhang. "왜 같은 모델·같은 시기에 어떤 팀은 throughput이 3배인가"의 답.
8. **Vibe Coding의 현장과 정형 검증 — 환경에 깔리는 도구들 (8a)** — 환각 카탈로그(6패턴), 슬롭스쿼팅 USENIX 2025, Cursor npm 404 사건, Devin Railway 사건, Claude Code Issue #10628 자가 환각 루프, 한국어 LLM 환각 정량(KT 믿음·HyperCLOVA·Kanana), 그리고 타입·LSP·컴파일러·TDD·sandbox·dry-run.
9. **자기 검증·외부 grounding·HITL — 모델과 시스템 설계의 패턴 (8b)** — CoVe, SelfCheckGPT, Semantic Entropy, verification subagent, Best-of-N, RAG, tool use, FACTS Grounding, Anthropic 공식 5가지 권장, 그리고 HITL.
10. **능력 과잉을 끌어내는 법 — 자동화·하네스·환각 자산화 (9장)** — 자동 검증 루프 다이어그램, parallel agents, prompt → context → harness의 3단계 진화, 환각을 자산화하는 brainstorming 워크플로, "내 하네스" 청사진.
11. **환각하는 기계와 함께 사는 법 — 네 관점, 한 자세 (10장)** — Amodei · OpenAI paper · Gary Marcus · Karpathy 네 관점 병기, "agentic coding은 함정이다" 비판의 정직한 수용, 함정을 피하는 다섯 가지 자세, 책을 덮자마자 작동시킬 첫 한 줄.

**Toby-AI**는 book-writer 하네스 위에서 작동하는 AI 저자 페르소나다. 리서치(web · paper · community) → 저술 계획 → 계획 리뷰 → 챕터 저술(토비 문체) → 스타일 리뷰 → 편집 → 표지 디자인 → EPUB 빌드의 8단계를 각각 독립된 에이전트와 컨텍스트에서 수행하고, 한 오케스트레이터가 그 흐름을 묶는다. 이 책은 그 하네스 v1.2.0으로 자동 산출되었다 — 9장에서 묘사한 *하네스의 한 구체적 모습*이 이 책 자체에서 작동한 셈이다.
