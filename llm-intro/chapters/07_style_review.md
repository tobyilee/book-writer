### S3. 7.3 기본값 감각 단락이 촘촘히 나열식

**원문 (139줄):**
> 실무 기본값에 대한 감을 짧게 짚자. `temperature=0.7` + `top_p=0.9` + `max_tokens`는 용도별로 명시. 이게 "일단 돌려보기" 디폴트다. 용도별 조정은 이렇게 간다. 분류·추출·요약처럼 **사실성과 재현성**이 중요하면 `temperature=0 ~ 0.2`. 창작·브레인스토밍처럼 **다양성**이 중요하면 `temperature=0.7 ~ 1.2`. 코드 생성은 보통 `0.0 ~ 0.3`. `top_p`는 대부분 0.9나 1.0을 써도 무방하다. …

[제안]
> 실무 기본값에 대한 감을 짧게 잡아두자. "일단 돌려보기" 디폴트는 `temperature=0.7`, `top_p=0.9`, `max_tokens`는 용도별로 명시다. 여기서 출발해서 용도에 맞게 조금씩 민다.
>
> | 용도 | `temperature` 권장 | 이유 |
> |---|---|---|
> | 분류·추출·요약 | 0 ~ 0.2 | 사실성·재현성 우선 |
> | 코드 생성 | 0.0 ~ 0.3 | 문법 실수 최소화 |
> | 창작·브레인스토밍 | 0.7 ~ 1.2 | 다양성이 값어치 |
>
> `top_p`는 대부분 0.9나 1.0을 써도 무방하다. 다만 `temperature`와 `top_p`를 둘 다 과격하게 흔들면 결과가 예측이 안 되니, **둘 중 하나만 주로 조절**하는 편이 낫다. OpenAI 문서도 같은 조언을 한다.

**이유:** 표로 뽑으면 (a) 나열식 압박이 풀리고, (b) 독자 복습용으로도 기능하고, (c) 분량도 약간 줄어든다. 7.3 절 자체가 "연결만 한다"는 약속이니 밀도 높은 대응표가 어울린다.

## 통과 지점

### 명시적 칭찬 9건

1. **7.1 여정 지도 (14줄 도식)** — 2장·4장 도입부와 같은 포맷으로 "지금 우리가 어디에 있나"가 또렷이 작동한다. "★ [7장 …] ← 지금 여기" 마크가 독자의 현재 위치감을 한 번에 잡아준다.
2. **7.2 오프닝 "대화의 모양으로 쓰인다"** — 개념 도입을 한 문장 선언으로 잡고 이어서 역사적 맥락을 풀어내는 서사가 2장 BPE 절의 톤과 같다.
3. **7.3 "설명이 아니라 연결"** — 절 도입부에서 장의 역할을 명시("개념은 거기서 이미 몸으로 잡았다")한 뒤 대응표로 바로 가는 흐름. 4장 디코딩 루프 복습으로 완벽히 기능한다.
4. **7.4 "백엔드 개발자에게 '외부 상태가 내 함수의 결과를 바꾼다'는 건 있을 수 없는 일이다."** — 한 문장이 장 전체 메타포를 박는다. 독자의 배경지식(순수 함수)을 불러와 충격을 극대화.
5. **7.4 네 전략** — seed / 모델 ID 고정 / 로컬 배치 1 / 의미 일치 테스트. 실무감이 날카롭고 `system_fingerprint`까지 짚는 디테일이 좋다.
6. **7.5 Few-shot 비용 경고** — "Few-shot을 한국어로 많이 깔면 입력 비용이 빠르게 불어난다. 짧게, 대표적으로." 2장 토크나이저 비용 논의와 수미상관.
7. **7.6 Anthropic SDK 대비 설명** — "OpenAI와 차이가 두 군데다. 첫째, `system`이 따로 빠져 있다. 둘째, JSON 강제 모드가 없으니…"의 두 지점 짚기가 군더더기 없다.
8. **7.7 회고 콜아웃** — "잠시 멈춤." 한 단어로 호흡을 끊고 "이 책의 한가운데를 지나고 있다"로 여는 전환이 메타 없이 서사적이다. 공통 태스크 4답 표가 9장 플로차트의 전조로 완벽하게 기능한다.
9. **7.10 Ollama 박스 — "머리로 고르지 말고 손으로 고르자"** — 5장·6장 철학과 수미상관. 로컬 모델을 "대안"이 아닌 "레파토리의 한 축"으로 자리매김시킨다.

### Python↔Java 2단 구조 일관성

- **태스크 모양:** 사내 개발 문서 요약 / JSON 출력 / 키는 `summary`·`keywords`·`confidence` / temperature 0.2 / max_tokens 512. → Python(OpenAI) → Python(Anthropic) → Python(LangChain) → Java(Spring AI) → Java(LangChain4j) 다섯 번 모두 **같은 의미론**을 유지한다. 특히 `Summary` POJO가 Spring AI `entity()`와 LangChain4j `AiServices` 양쪽에서 재등장하는 장면은 독자에게 "같은 태스크가 다른 도구로 재현된다"는 감각을 분명히 심는다.
- **provider 스왑 약속의 수미상관:** 7.8 Spring AI pom 한 줄 스왑 → 7.9 LangChain4j builder 스왑 → 7.10 Ollama base_url 스왑. 세 번 모두 "의존성 한 줄" 약속이 지켜졌고, "같은 SummaryService 코드가 OpenAI, Anthropic, Ollama 위에서 모두 동작한다"(676줄)가 장 전체 약속의 감정 피크.
- **Spring AI vs LangChain4j가 경쟁 아닌 대안:** 7.9 686줄 "둘 다 한 번 써보고 손에 맞는 걸 고르는 편이 낫다"가 Toby 권장 어조의 정석.
