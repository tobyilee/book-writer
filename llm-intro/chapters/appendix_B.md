# 부록 B. 용어집 — 본문 역인덱스 포함

책 본문에서 만난 용어를 한자리에 모았다. 가나다 순으로 보고 싶을 때, 영문으로 찾고 싶을 때, 둘 다 열쇠다. 각 용어 옆에 **어느 장에서 처음 만나는지** 숫자가 박혀 있으니 다시 펴볼 때 길잡이로 쓰자. 정의는 한 줄, 길어야 두 줄로 담았다. 더 깊이 들어가고 싶으면 해당 장으로 돌아가면 된다.

## B.1 가나다 순 용어집

- **가중치 (Weights) — [3장, 4장]** — 어텐션이 토큰 간 참조 강도를 담는 숫자 행렬. 학습은 결국 이 숫자들을 고쳐 쓰는 과정이다.
- **감사 로그 (Audit Log) — [9장]** — 모델의 입출력과 의사결정 근거를 시간순으로 남기는 기록. 사고 재연과 규제 대응의 1차 자료.
- **관측성 (Observability) — [7장, 9장]** — 프롬프트·응답·지연·토큰 사용량을 수집해 운영 중인 LLM을 들여다보는 실천. LangSmith·Langfuse·Spring AI Observation이 대표 도구.
- **가드레일 (Guardrails) — [9장]** — 모델의 입출력 경계에서 위험한 요청·응답을 걸러내는 장치. NeMo Guardrails·Llama Guard 등.
- **그리디 디코딩 (Greedy Decoding) — [4장]** — 매 스텝에서 확률이 가장 높은 토큰 하나만 고르는 가장 단순한 생성 전략. 반복이 잘 나온다.
- **나노지피티 (nanoGPT / build-nanogpt / nanochat) — [4장]** — Karpathy가 공개한 교육용 미니 GPT 저장소 계보. 4장 실습의 뼈대.
- **뉴클리어스 샘플링 (Nucleus Sampling, Top-p) — [4장, 7장]** — 누적 확률 p 안쪽의 토큰만 추려 그 안에서 뽑는 디코딩 전략. 자연스러운 생성의 기본값.
- **다중 헤드 어텐션 (Multi-Head Attention) — [3장]** — 여러 시점의 Q/K/V를 동시에 계산해 서로 다른 관계를 포착하는 구조.
- **디코더 온리 (Decoder-only) — [3장]** — 마스크드 셀프 어텐션만 쌓아 올려 다음 토큰을 예측하는 현대 LLM의 표준 아키텍처.
- **디코딩 루프 (Decoding Loop) — [4장, 7장]** — `logits → softmax → sampling` 을 반복해 토큰을 한 개씩 이어 붙이는 루프. 4장에서 손으로 짜고 7장에서 API 손잡이로 다시 만난다.
- **레이어 정규화 (LayerNorm) — [3장]** — 각 층 출력의 분포를 맞춰 학습을 안정화하는 층. Residual과 한 짝처럼 붙는다.
- **레즈듀얼 연결 (Residual Connection) — [3장]** — 입력을 출력에 더해 그라디언트가 쉽게 흐르게 하는 지름길. 깊은 네트워크의 필수 장치.
- **로라 (LoRA, Low-Rank Adaptation) — [6장]** — 원본 가중치는 두고 저랭크 행렬 두 개만 학습해 붙이는 파라미터 효율 기법.
- **리랭커 (Reranker) — [8장]** — 1차 검색 결과를 문맥 적합도로 재정렬하는 후처리 모델. 정확도와 비용의 교환.
- **마스크드 어텐션 (Masked Attention) — [3장]** — 미래 토큰을 못 보도록 차단하는 어텐션. 디코더 온리의 핵심.
- **매개변수 효율 미세조정 (PEFT, Parameter-Efficient Fine-Tuning) — [6장]** — 극히 일부 파라미터만 학습해 비용을 줄이는 계열의 총칭. LoRA·QLoRA가 대표.
- **모델 컨텍스트 프로토콜 (MCP, Model Context Protocol) — [10장]** — 모델과 외부 도구·데이터 소스를 잇는 개방 표준. 에이전트 시대의 배선도.
- **문맥 내 학습 (In-Context Learning) — [5장]** — 별도 학습 없이 프롬프트 안의 예시만으로 새 태스크를 흉내 내는 능력. GPT-3의 발견.
- **바이그램 (Bigram) — [4장]** — 바로 앞 토큰 하나만 보고 다음을 예측하는 최소 모델. 4장 학습 곡선의 출발선.
- **바이트 쌍 인코딩 (BPE, Byte-Pair Encoding) — [2장]** — 자주 등장하는 바이트 쌍을 반복 병합해 서브워드 어휘를 만드는 토큰화 알고리즘.
- **베이스·인스트럭트·챗 모델 (Base / Instruct / Chat) — [5장]** — 같은 가중치가 Pretraining·Instruction Tuning·RLHF 단계를 거치며 변해 간 세 얼굴.
- **벡터 데이터베이스 (Vector DB) — [8장]** — 임베딩을 저장하고 유사도로 검색하는 저장소. pgvector·Chroma·Qdrant.
- **샘플링 (Sampling) — [4장, 7장]** — 확률 분포에서 다음 토큰을 뽑는 행위 전반. Greedy·Temperature·Top-k·Top-p가 그 손잡이.
- **사전학습 (Pretraining) — [5장]** — 방대한 텍스트로 "다음 토큰 예측"만 반복해 기초 지식을 쌓는 단계. 현대 LLM의 토대.
- **사전학습·지도학습·선호학습 3단계 (SFT → RM → PPO) — [5장]** — RLHF의 표준 파이프라인. 지시 튜닝 → 보상 모델 학습 → PPO로 정책 개선.
- **서브워드 (Subword) — [2장]** — 단어도 문자도 아닌 그 사이 단위. 희귀어·합성어를 조각으로 처리한다.
- **셀프 어텐션 (Self-Attention) — [3장]** — 같은 시퀀스 안의 토큰들끼리 서로를 참조하는 어텐션. 트랜스포머의 심장.
- **소프트맥스 (Softmax) — [3장, 4장]** — 실수 벡터를 합이 1인 확률 분포로 바꾸는 함수. 어텐션 가중치와 다음 토큰 분포가 여기서 나온다.
- **스케일드 닷 프로덕트 어텐션 (Scaled Dot-Product Attention) — [3장]** — Q·K 내적을 √d로 나눠 안정화한 뒤 softmax로 가중치를 만드는 어텐션의 표준 수식.
- **스케일링 법칙 (Scaling Laws) — [5장]** — 파라미터·데이터·연산이 커질수록 손실이 거듭제곱으로 떨어진다는 경험칙. Kaplan → Chinchilla → Llama의 3막.
- **스트리밍 (Streaming / SSE) — [7장]** — 생성된 토큰을 즉시 클라이언트로 흘려보내는 방식. 체감 지연을 줄인다.
- **시스템·유저·어시스턴트 (system / user / assistant) — [7장]** — Chat Completions API의 세 가지 역할. 지시·입력·응답을 구조화한다.
- **양자화 (Quantization) — [6장, 10장]** — 가중치 정밀도를 16비트 또는 4비트로 줄여 메모리와 속도를 얻는 기법. NF4·GPTQ·AWQ가 주요 방식.
- **어텐션 (Attention) — [3장]** — 토큰이 다른 토큰을 얼마나 참조할지 정하는 가중치 계산. 3장의 심장.
- **어휘사전 (Vocabulary) — [2장]** — 토크나이저가 아는 모든 토큰의 목록과 ID 대응표. 크기가 곧 임베딩 테이블 행 수.
- **에이전트 (Agent) — [10장]** — LLM이 도구 호출·관찰·재계획을 반복해 다단계 과제를 수행하는 구조. 2025~2026 현장의 기본값.
- **에폭 (Epoch) — [4장]** — 훈련 데이터 전체를 한 번 돌린 단위. 너무 많이 돌리면 과적합이 온다.
- **엠베딩 (Embedding) — [2장, 8장]** — 토큰 또는 문장을 고정 차원의 실수 벡터로 바꾼 표현. 의미가 가까우면 벡터도 가깝다.
- **엠피에스 / 쿠다 (MPS / CUDA) — [4장]** — Apple Silicon GPU와 NVIDIA GPU 각각의 PyTorch 백엔드. `torch.device("mps")` 또는 `cuda`로 스위치.
- **역량 출현 (Emergent Ability) — [5장]** — 규모가 임계를 넘어설 때 예고 없이 나타나는 능력. 스케일링 서사의 수수께끼.
- **온도 (Temperature) — [4장, 7장]** — softmax의 날카로움을 조절하는 파라미터. 높이면 다양성, 낮추면 결정성.
- **오픈 모델 (Open Model) — [6장, 7장]** — 가중치가 공개된 LLM(Llama·Gemma·Qwen 등). QLoRA·Ollama 실습의 주역.
- **올라마 (Ollama) — [7장, 10장]** — 로컬에서 오픈 모델을 API로 띄우는 런타임. Mac에서 `bge-m3`·Llama 3.1 8B 구동의 기본 경로.
- **위치 인코딩 (Positional Encoding) — [3장]** — 어텐션에 순서 정보를 주입하는 사인·코사인 기반 벡터. RoPE·YaRN은 그 진화형.
- **임베딩 모델 (Embedding Model) — [8장]** — 문장을 RAG용 벡터로 변환하는 모델. `bge-m3`, OpenAI `text-embedding-3-small`.
- **정책 그라디언트 (PPO, Proximal Policy Optimization) — [5장]** — RLHF 3단계의 마지막. 보상 모델 점수를 올리도록 정책을 미세조정한다.
- **지시 튜닝 (Instruction Tuning / SFT) — [5장]** — "지시에 답하는" 양식의 데이터로 Base 모델을 지도학습하는 단계. Base를 Instruct로 만든다.
- **질의·키·값 (Q / K / V) — [3장]** — 어텐션이 토큰마다 만드는 세 벡터. Q는 질문, K는 간판, V는 선반 내용이다.
- **청킹 (Chunking) — [8장]** — 긴 문서를 검색 가능한 조각으로 자르는 과정. 너무 작거나 크면 RAG 품질이 바로 무너진다.
- **챗클라이언트 (ChatClient) — [7장]** — Spring AI가 제공하는 공용 LLM 호출 인터페이스. provider 스왑을 한 줄로.
- **크로스 엔트로피 손실 (Cross-Entropy Loss) — [4장]** — 다음 토큰 예측 분포와 정답 분포의 거리. LLM 학습의 기본 목적 함수.
- **탑케이 샘플링 (Top-k Sampling) — [4장]** — 상위 k개 토큰만 남기고 그 안에서 뽑는 디코딩 전략. Top-p와 조합해 쓰기도 한다.
- **텐션 (Attention, 재지정 생략)** → 어텐션 참조.
- **토큰 (Token) — [2장]** — 모델이 실제로 먹는 최소 단위의 정수. "문자 덩어리"가 아니라 "정해진 조각".
- **토큰화 (Tokenization) — [2장]** — 문자열을 어휘사전에 따라 정수 시퀀스로 쪼개는 과정. BPE·WordPiece·SentencePiece 계보.
- **트랜스포머 (Transformer) — [3장]** — 어텐션과 MLP, 레지듀얼·LayerNorm을 쌓아 만든 아키텍처. Vaswani 외(2017) 논문에서 출발.
- **티티에프티 / 티시오 (TTFT / TCO) — [9장]** — Time To First Token과 Total Cost of Ownership. 지연·비용 판단의 두 축.
- **파인튜닝 (Fine-tuning) — [5장, 6장]** — 사전학습된 가중치를 특정 태스크에 맞춰 추가 학습하는 모든 작업의 총칭.
- **프롬프트 인젝션 (Prompt Injection) — [9장]** — 입력 문자열로 시스템 지시를 탈취하려는 공격. 1순위 위협.
- **피드포워드 (Feed-Forward, FFN) — [3장]** — 어텐션 뒤에 붙는 2층 MLP. 토큰별로 독립 적용된다.
- **함수 호출 (Function Calling / Tool Use) — [7장, 10장]** — 모델이 JSON 스키마에 맞춰 외부 함수를 호출하도록 유도하는 기능. 에이전트의 기본 단위.
- **허깅페이스 (Hugging Face) — [6장]** — 모델·데이터셋·스페이스를 공유하는 허브. `transformers`·`peft`·`trl`이 6장 실습의 주 도구.
- **하이브리드 검색 (Hybrid Search) — [8장]** — 키워드(BM25)와 임베딩 검색을 결합한 전략. 한쪽만 쓸 때 놓치는 리콜을 메운다.
