# 03 리뷰 로그 — 02_plan.md

## 2026-05-25 · 1차 계획 리뷰 반영 (plan v1 → v2)

리뷰 피드백을 항목별로 반영했다. 형식: 피드백 → 반영 내역 → 변경 위치.

### 반드시 반영 (4건)

**1. 4~7장 도입형 4연속 피로 — 6·7장에 before/after 문제-해결형 사례 의무 배치 또는 통합**
- 반영: 기존 6장(spec-driven)+7장(harness)을 한 장(신6장 "큰 작업을 구조화하기")으로 통합하고 유형을 "문제-해결형 + 사례 분석형"으로 바꿈. 신6장 오프닝을 before/after(모호한 프롬프트→명세 명시) 문제-해결형으로 진입하도록 의무화. 5장에도 찜찜한 컨텍스트→개선 before/after를 장의 중심 산출물로 격상. 결과적으로 도입형 사다리가 4~6장 3칸으로 줄고, 6장은 문제-해결형으로 전환되어 도입형 연속 피로 해소.
- 위치: 챕터 목록 신6장 전면 개편, 신5장 산출물 항목.

**2. 7장(harness) vs 5장(context) 위계 중복 — 통합 권장**
- 반영: 권장안(통합) 채택. harness가 "prompt < context < harness"로 context의 상위 확장이자 spec의 운영 골격임을 통합 근거로 명시. 7장을 6장(spec-driven)과 묶어 "큰 작업을 구조화하기 — 명세와 하네스" 단일 장으로 통합. **책이 11장 → 10장으로 축소.** 분량은 신6장 8,500자로 잡아 두 주제를 압축 수용.
- 위치: 책 특성(분량 11장→10장), 내러티브 아크(4~6장 3단 사다리로 재서술), 챕터 목록(신6장), 이후 8·9·10·11장 → 7·8·9·10장 전면 재번호.

**3. 70/30 무게중심 정합성 — 성찰 단락 무게 명시, 목표 25~30% 못 박기**
- 반영: 책 특성에 "실전 기법 70~75% / 사색·성장 25~30%"로 목표를 솔직히 명기. 각 실전 장의 "성찰"이 한 문장 장식이 아니라 **최소 한 단락(한글 200~300자)** 분량을 갖고, 1장 두려움·닫는 장 성장 서사와 명시적으로 잇도록 집필 의무 지침을 신설. 모든 장의 "짧은 성찰"을 "성찰(한 단락, 200~300자)"로 라벨 교체.
- 위치: 책 특성(무게중심·성찰 무게 항목 신설), 전 챕터 성찰 항목.

**4. 7·10·11장 추상론 방지 — 복붙 가능한 산출물 각 1개 의무화**
- 반영: 책 특성에 "복붙 가능한 산출물" 의무 항목 신설. 구조화 장(신6장)=핸드오프 문서 양식 + 페이즈 게이트 체크리스트, 신9장(팀)=팀 검증 문화 제안 1페이지 템플릿 + PR당 검증 체크리스트, 신10장(성장)=개인 성장 루프 요약 카드(위임→검증→학습→더 나은 위임). 추가로 신6장에 도구 선택 기준 매트릭스, 신8장에 의도적 연습 시간블록 처방까지 산출물로 명시.
- 위치: 책 특성(산출물 항목), 신6·8·9·10장 "핵심 산출물" 항목.
